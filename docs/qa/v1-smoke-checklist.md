# Deskmon V1 Smoke Checklist

用途：每次准备发一个可试用版本前，手工确认 V1 主流程没有退化。

## 准备

- 关闭旧的 Deskmon 进程。
- 启动最新构建的 `Deskmon.app`。
- 确认 macOS 通知权限允许 Deskmon 发送通知。
- 如果要覆盖显示器变化，准备一次外接屏连接或断开。

## 宠物窗口

- 启动后能看到透明无边框的像素史莱姆窗口。
- 史莱姆只在可见工作区内出现，不被左侧 Dock 遮住。
- 左键单击史莱姆没有任何动作。
- 左键拖拽不会跳到鼠标中心，松手后停在释放位置附近。
- 鼠标悬停在史莱姆上时，它停止自主移动，但动画仍在播放。
- 鼠标离开后，史莱姆会恢复自主移动或休息状态。

## 动画节奏

- `idle` 看起来安静但仍有轻微生命感，没有明显抖动。
- `walk` 的播放速度和窗口移动速度匹配，不像贴图平移滑行。
- `run` 明显比 `walk` 更有冲劲，但不会闪得不可读或乱窜。
- `sleep` 保持慢和安静，适合长时间停在桌面上。
- `timer-waiting` 像是在耐心等待，不显得焦虑。
- `dragged` 保留被拎起来的趣味，松手后不会突兀跳状态。
- `celebrate` 保持活泼，计时完成和巨型庆祝期间都不会过度闪烁。
- 安静 / 标准 / 活泼三档存在感有明显差异：安静更少动，标准最自然，活泼更有精神但不烦。

## 菜单栏菜单

- 菜单栏图标可打开原生菜单。
- “隐藏宠物”会隐藏宠物窗口，菜单栏仍可用。
- “显示宠物”会恢复宠物窗口。
- “暂停移动”后宠物不再改变位置，但仍有安静动画。
- “恢复移动”后宠物会重新自主移动。
- “移回活动区域”会显示宠物，并把它移回当前活动区域内。
- “设置”能打开设置窗口。
- “退出”能退出应用。

## 宠物小菜单

- 右键史莱姆打开 macOS 原生小菜单。
- 小菜单未计时时只包含计时器、暂停/恢复移动、隐藏。
- 小菜单不包含设置。
- 小菜单不包含退出。

## 计时器

- 菜单栏未计时时，“专注计时”子菜单包含专注 5 / 25 / 45 分钟和休息 5 分钟。
- 宠物小菜单未计时时，“专注计时”子菜单包含专注 5 / 25 / 45 分钟和休息 5 分钟。
- 启动专注计时后，两个菜单都显示“专注中：还剩 MM:SS”和取消入口。
- 启动休息计时后，两个菜单都显示“休息中：还剩 MM:SS”和取消入口。
- 计时中菜单不再显示新的开始计时入口。
- 打开菜单时，剩余时间会持续刷新，菜单不会因为刷新自动关闭。
- “取消当前计时”能让宠物退出等待状态。
- 专注自然结束后，macOS 通知正文使用专注结束提示文案。
- 休息自然结束后，macOS 通知正文使用休息结束提示文案；声音开关开启时播放系统通知声音。
- 宠物可见时，计时完成会进入巨型庆祝动效：前 3 秒边变大边移动，3s 到 6s 原地庆祝，6s 到 7s 缩回并回到原位。
- 巨型庆祝动效期间，史莱姆全程播放 `celebrate`，不显示额外桌面文字。
- 巨型庆祝动效显示在普通窗口上方，但不会抢走当前应用的键盘焦点。
- 巨型庆祝动效期间，左键拖拽和右键小菜单不会触发。
- 巨型庆祝动效期间，从菜单栏启动新计时器会结束庆祝并让宠物进入等待形态。
- 巨型庆祝动效期间，从菜单栏隐藏宠物会让宠物恢复普通尺寸/位置后隐藏。
- 宠物隐藏时，计时完成只发送通知，不自动显示宠物。
- 拖拽宠物时倒计时自然结束，只发送通知，不触发巨型庆祝。

## 设置窗口

- 尺寸切换为小 / 中 / 大后，宠物窗口大小即时变化。
- 活跃程度切换为安静 / 标准 / 活泼后，移动频率和速度有明显差异。
- 默认置顶开关能即时影响宠物窗口层级。
- 专注计时区域能修改 3 个专注时长、休息时长、两条完成提示文案和休息结束声音开关。
- 专注时长重复或超出 1-180 分钟时显示错误，不保存。
- 专注时长保存后按从小到大展示，重新打开设置仍然保留。
- 活动区域框选拖动时被限制在默认活动区域内。
- 活动区域太小时显示警告，不会保存。
- 活动区域足够大时能保存，并且宠物之后只在该区域内自主移动。
- “重置默认”会清除自定义活动区域。

## 恢复能力

- 把宠物拖到活动区域外但仍在可见工作区内，宠物会逐步回到活动区域。
- 重启应用后，宠物恢复到上次可见位置。
- 如果显示器变化导致上次位置不可见，宠物会回到可见区域。
- 菜单栏“移回活动区域”能兜底找回宠物。
- 巨型庆祝动效结束后，宠物尺寸、位置和置顶状态恢复正常，不把巨型尺寸或临时位置保存成偏好。

## 长时间运行

- 运行 30 分钟后，宠物仍会移动、休息和响应菜单。
- 空闲时 CPU 占用没有明显持续飙高。
- 内存没有持续增长。
- 计时器在长时间运行后仍能启动、取消和完成。
