# Deskmon V1 Smoke Checklist

用途：每次准备发一个可试用版本前，手工确认 V1 主流程没有退化。

## 准备

- 关闭旧的 Deskmon 进程。
- 启动最新构建的 `Deskmon.app`。
- 确认 macOS 通知权限允许 Deskmon 发送通知。
- 如果要覆盖显示器变化，准备一次外接屏连接或断开。

## 宠物窗口

- 启动后能看到透明无边框的像素史莱姆窗口。
- 史莱姆只在可见工作区内出现，不被左侧 Dock 遮住。
- 左键单击史莱姆没有任何动作。
- 左键拖拽不会跳到鼠标中心，松手后停在释放位置附近。
- 鼠标悬停在史莱姆上时，它停止自主移动，但动画仍在播放。
- 鼠标离开后，史莱姆会恢复自主移动或休息状态。

## 动画节奏

- `idle` 看起来安静但仍有轻微生命感，没有明显抖动。
- `walk` 的播放速度和窗口移动速度匹配，不像贴图平移滑行。
- `run` 明显比 `walk` 更有冲劲，但不会闪得不可读或乱窜。
- `sleep` 保持慢和安静，适合长时间停在桌面上。
- `timer-waiting` 像是在耐心等待，不显得焦虑。
- `dragged` 保留被拎起来的趣味，松手后不会突兀跳状态。
- `celebrate` 保持活泼，计时完成和巨型庆祝期间都不会过度闪烁。
- 安静 / 标准 / 活泼三档存在感有明显差异：安静更少动，标准最自然，活泼更有精神但不烦。

## 菜单栏菜单

- 菜单栏图标可打开原生菜单。
- “隐藏宠物”会隐藏宠物窗口，菜单栏仍可用。
- “显示宠物”会恢复宠物窗口。
- “暂停移动”后宠物不再改变位置，但仍有安静动画。
- “恢复移动”后宠物会重新自主移动。
- “移回活动区域”会显示宠物，并把它移回当前活动区域内。
- “设置”能打开设置窗口。
- “退出”能退出应用。

## 宠物小菜单

- 右键史莱姆打开 macOS 原生小菜单。
- 小菜单未计时时只包含计时器、暂停/恢复移动、隐藏。
- 小菜单不包含设置。
- 小菜单不包含退出。

## 计时器

- 菜单栏和宠物小菜单未计时时，“专注计时”只包含专注 5 / 25 / 45 分钟，没有独立休息入口。
- 启动专注后，两个菜单都显示“专注中：还剩 MM:SS”和“取消本轮”。
- 专注中菜单不再显示新的开始计时入口。
- 打开菜单时，剩余时间会持续刷新，菜单不会因为刷新自动关闭。
- “取消本轮”立即恢复普通桌宠，不发送通知、不播放声音、不庆祝。
- 专注自然结束后，通知使用专注提示文案；宠物移动到鼠标所在屏幕，放大并播放一轮庆祝。
- 放大完成后，宠物保持居中，旁边显示“休息 X 分钟 / 再专注 5 分钟 / 结束本轮”。
- 完成对话框不会超时、点击外部不会关闭，也不会抢走当前应用的键盘焦点。
- 选择“休息 X 分钟”后，宠物保持放大居中，显示“休息中 MM:SS”和“提前结束休息”。
- 休息中两个原生菜单显示剩余时间和“提前结束休息”，不显示普通专注入口。
- 提前结束休息不发送通知、不播放声音、不庆祝，直接进入休息完成选择。
- 休息自然结束后，通知使用休息提示文案；声音开启时播放系统声音，并播放一轮庆祝。
- 休息完成后显示“继续专注 X 分钟 / 再休息 5 分钟 / 结束本轮”。
- “再专注 5 分钟”和“再休息 5 分钟”可以重复，且不会覆盖本轮原始专注和休息时长。
- 选择继续专注或结束本轮后，宠物恢复原大小、位置、置顶状态和可见性。
- 宠物隐藏时专注到点仍临时显示中央交接；重新专注或结束后恢复隐藏。
- 拖拽宠物时专注到点不会抢走拖拽，松手后立即进入中央交接。
- 中央状态禁止拖拽，但右键菜单可用，且操作与旁边的对话框一致。
- 中央状态中选择“结束本轮并隐藏宠物”会停止当前流程并隐藏宠物。
- 对话框获焦后，方向键可切换按钮，Enter/Space 可执行，Escape 不关闭。

## 设置窗口

- 尺寸切换为小 / 中 / 大后，宠物窗口大小即时变化。
- 活跃程度切换为安静 / 标准 / 活泼后，移动频率和速度有明显差异。
- 默认置顶开关能即时影响宠物窗口层级。
- 专注计时区域能修改 3 个专注时长、休息时长、两条完成提示文案和休息结束声音开关。
- 专注时长重复或超出 1-180 分钟时显示错误，不保存。
- 专注时长保存后按从小到大展示，重新打开设置仍然保留。
- 区域截图区域显示当前有效保存目录，且位于专注计时和活动区域之间。
- “选择文件夹”使用原生目录选择器，保存后重新打开设置仍然保留。
- “恢复默认”清除自定义目录并重新显示桌面路径。
- 活动区域框选拖动时被限制在默认活动区域内。
- 活动区域太小时显示警告，不会保存。
- 活动区域足够大时能保存，并且宠物之后只在该区域内自主移动。
- “重置默认”会清除自定义活动区域。

## 区域截图

- 菜单栏和宠物右键菜单都有一级“区域截图”入口，两个入口启动同一套系统框选。
- 启动后只能拖拽矩形区域，按空格不会切换成窗口截图。
- 松开鼠标后生成 PNG，文件名为 `Deskmon_YYYY-MM-DD_HH-mm-ss.png`。
- 同一秒存在同名文件时使用 `_2`、`_3` 序号，不覆盖已有文件。
- 截图不包含鼠标，完成时保留系统截图声音，并发送包含文件名的成功通知。
- 按 `Escape` 取消后不生成文件、不播放声音，也不发送通知。
- 截图不会写入剪贴板，也不会进入记忆中、笔记本或变量库。
- 自定义目录不存在时会自动创建；目录不可用时先提示重新选择或恢复桌面。
- 缺少屏幕录制权限时显示说明，并只在确认后打开系统设置。
- 截图期间宠物继续活动，专注计时、设置窗口、记忆力窗口和中央交接不改变。
- 框选到 Deskmon 时，宠物或窗口正常出现在截图中。
- 截图进行中再次触发入口不会叠加第二个框选层。

## 恢复能力

- 把宠物拖到活动区域外但仍在可见工作区内，宠物会逐步回到活动区域。
- 重启应用后，宠物恢复到上次可见位置。
- 如果显示器变化导致上次位置不可见，宠物会回到可见区域。
- 菜单栏“移回活动区域”能兜底找回宠物。
- 继续专注或结束本轮后，宠物尺寸、位置和置顶状态恢复正常，不把中央呈现写入偏好。

## 长时间运行

- 运行 30 分钟后，宠物仍会移动、休息和响应菜单。
- 空闲时 CPU 占用没有明显持续飙高。
- 内存没有持续增长。
- 计时器在长时间运行后仍能启动、取消和完成。
