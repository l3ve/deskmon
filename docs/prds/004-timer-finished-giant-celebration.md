# PRD：Deskmon 倒计时结束巨型庆祝提醒

状态：需求已通过 grill-me / grill-with-docs 收敛，待实现拆分
目标 triage 标签：ready-for-agent
创建日期：2026-07-06
最后更新：2026-07-06
发布状态：已发布到 GitHub issue #5：https://github.com/l3ve/deskmon/issues/5

## Problem Statement

用户已经可以通过 Deskmon 启动固定时长倒计时。倒计时进行中，史莱姆会进入等待形态；倒计时结束时，应用会发送 macOS 通知，并让史莱姆短暂切换到庆祝形态。

问题是当前完成反馈不够显眼。史莱姆仍然保持日常小尺寸，如果用户正专注在其他窗口、视线没有落在宠物附近，可能只注意到系统通知，而没有感受到“桌面宠物完成提醒”的存在感。用户希望倒计时结束时，史莱姆能更明显地提醒自己，比如临时变得很大，而不是只换一个小小的形态。

这个增强仍然要保持 Deskmon 的克制边界。它不是全屏 overlay，不抢焦点，不加入音效，不把倒计时扩展成完整提醒系统，也不违背“隐藏宠物就不要出现在桌面上”的语义。

## Solution

当倒计时结束且宠物当前可见、没有被用户拖拽时，Deskmon 进入一个短暂的“巨型庆祝提醒”状态。史莱姆临时变大到当前所在屏幕可用工作区的醒目尺寸，移动到当前屏幕内合适位置，临时置顶播放庆祝动画。提醒期间不抢焦点，不响应拖拽或右键菜单。

这 7 秒被定义为“巨型庆祝动效”，分为三段：

- 0s 到 3s 是进入段：史莱姆持续播放 `celebrate`，窗口按中心点从倒计时结束瞬间的位置和尺寸，平滑变大并移动到巨型提醒目标位置。
- 3s 到 6s 是停留段：史莱姆保持巨型尺寸和提醒位置，原地持续播放 `celebrate`。
- 6s 到 7s 是恢复段：史莱姆继续播放 `celebrate`，窗口按中心点平滑缩回普通展示尺寸，并移动回原宠物位置。

进入段和恢复段都使用直线路径，不做弧线、绕路、弹跳或物理模拟。进入段使用 ease-out，让提醒尽快显眼但收尾柔和；恢复段使用 ease-in-out，让回位不突兀。恢复完成后，宠物回到普通状态。

如果倒计时结束时宠物隐藏，保持现有隐藏语义：只发送 macOS 通知，不自动显示宠物，不触发巨型庆祝。如果倒计时结束时用户正在拖拽宠物，也和隐藏逻辑保持一致：只发送 macOS 通知，不打断拖拽，不在松手后补触发巨型庆祝。

如果用户在巨型庆祝动效期间从菜单栏隐藏宠物，隐藏命令优先，动效立即中断；Deskmon 先恢复普通尺寸和位置，再隐藏宠物。如果用户在动效期间改了宠物尺寸或置顶设置，恢复时尊重最新设置：尺寸和置顶状态使用最新偏好，位置以倒计时结束瞬间捕获的原宠物中心为基准并按新尺寸 clamp 到当前可见工作区。如果屏幕环境在动效期间变化，动效不需要中途重算目标；恢复结束时重新按当前屏幕环境 clamp，保证宠物可见。

巨型庆祝提醒不在宠物窗口中显示文字。提醒主角是史莱姆本体的放大庆祝，文字信息继续由 macOS 通知承担。

## User Stories

1. 作为 macOS 用户，我想倒计时结束时史莱姆变得明显可见，以便我不容易错过完成时刻。
2. 作为 macOS 用户，我想倒计时完成反馈像宠物在庆祝，以便计时器能力更像 Deskmon 桌宠体验的一部分。
3. 作为 macOS 用户，我想史莱姆能临时变大，以便完成反馈比小尺寸形态切换更醒目。
4. 作为 macOS 用户，我想巨型庆祝只持续很短时间，以便它能提醒我但不会一直挡住桌面。
5. 作为 macOS 用户，我想巨型庆祝自动恢复，以便我不需要手动把宠物缩小或移回去。
6. 作为 macOS 用户，我想庆祝结束后宠物回到原来的位置，以便桌面布局不被临时提醒打乱。
7. 作为 macOS 用户，我想庆祝结束后宠物回到普通展示尺寸，以便我的尺寸偏好仍然有效。
8. 作为 macOS 用户，我想庆祝结束后宠物恢复原来的置顶行为，以便临时提醒不会永久修改设置。
9. 作为 macOS 用户，我想巨型庆祝出现在宠物当前所在屏幕，以便多显示器时它不会跳到另一个屏幕。
10. 作为多显示器 macOS 用户，我想提醒遵守当前屏幕可用工作区，以便它不会出现在 Dock 后面或屏幕外。
11. 作为 macOS 用户，我想巨型提醒尺寸根据屏幕自适应，以便笔记本屏幕上足够醒目、大屏幕上也不过分夸张。
12. 作为 macOS 用户，我想提醒位置根据巨型宠物尺寸决定，以便较小提醒可以稍微偏下，较大提醒能更安全地接近中心。
13. 作为 macOS 用户，我想提醒显示在普通窗口上方，以便它不会被当前应用盖住。
14. 作为 macOS 用户，我不想提醒抢走焦点，以便我当前的输入或工作流程不会被打断。
15. 作为 macOS 用户，我想保留 macOS 通知，以便仍然有系统级完成提示。
16. 作为 macOS 用户，我想宠物隐藏时倒计时完成仍然保持隐藏，以便隐藏模式是可信的安静状态。
17. 作为 macOS 用户，我想拖拽宠物时不要突然变大，以便我正在控制宠物时窗口不会突然变形。
18. 作为 macOS 用户，我想拖拽中倒计时结束时只发送通知，以便 Deskmon 不打断我的拖拽手势。
19. 作为 macOS 用户，我想巨型庆祝期间不可交互，以便拖拽、菜单和位置保存不会产生歧义。
20. 作为 macOS 用户，我想巨型提醒期间禁用右键菜单，以便这个提醒保持简单的临时状态。
21. 作为 macOS 用户，我想在庆祝期间启动新计时器时停止旧庆祝，以便宠物立即反映新的计时任务。
22. 作为 macOS 用户，我想庆祝期间启动新计时器后宠物进入等待形态，以便当前状态清晰。
23. 作为 macOS 用户，我不想宠物窗口里额外显示桌面文字，以便提醒保持像素宠物的视觉调性。
24. 作为 macOS 用户，我不想这个功能新增声音，以便 Deskmon 继续保持安静。
25. 作为 macOS 用户，我不想第一版新增设置开关，以便功能默认可用且配置不变复杂。
26. 作为 macOS 用户，我不想出现全屏透明 overlay，以便 Deskmon 仍然是小桌宠而不是屏幕遮罩应用。
27. 作为 macOS 用户，我想即使动画循环持续运行，提醒也能恢复，以便宠物不会卡在巨型状态。
28. 作为 macOS 用户，我想取消计时器仍然保持原行为，以便只有自然完成的计时才触发庆祝。
29. 作为开发者，我想把触发策略和原生窗口控制分开，以便隐藏、拖拽、新计时和超时规则可以独立测试。
30. 作为开发者，我想原生窗口的临时变大和恢复有明确接口，以便临时提醒状态不会泄漏到持久偏好里。
31. 作为开发者，我想巨型庆祝避免写入持久设置，以便临时提醒不会改变宠物尺寸或最后位置。
32. 作为开发者，我想当前显示器和工作区计算集中处理，以便多显示器行为可预测。
33. 作为开发者，我想前端宠物状态能保护庆祝期间的指针交互，以便拖拽和菜单事件不会和临时尺寸冲突。
34. 作为开发者，我想为庆祝策略写外部行为测试，以便后续改计时器时不回归提醒逻辑。
35. 作为开发者，我想用 smoke test 覆盖系统级焦点和位置行为，以便 macOS 特有行为能在真实应用里验证。
36. 作为 macOS 用户，我想史莱姆在前 3 秒边变大边移动，以便提醒像动画而不是瞬间跳到位置。
37. 作为 macOS 用户，我想史莱姆在 3s 到 6s 保持巨型状态，以便我有足够时间注意到倒计时完成。
38. 作为 macOS 用户，我想史莱姆在 6s 到 7s 缩回并移动回原位，以便提醒退出得更柔和。
39. 作为 macOS 用户，我想三个动画阶段都保持庆祝动作，以便宠物在整个提醒期间都像在开心庆祝。
40. 作为 macOS 用户，我想动画按史莱姆中心点移动，以便放大和缩小时视觉上更自然。
41. 作为 macOS 用户，我想动效期间点击隐藏能立即生效，以便我的直接控制优先。
42. 作为 macOS 用户，我想动效期间的设置修改在恢复后被尊重，以便尺寸调整不会被旧状态覆盖。
43. 作为 macOS 用户，我想动效期间屏幕变化后仍能安全恢复，以便显示器或工作区变化后宠物仍然可见。
44. 作为开发者，我想动画时间轴明确，以便进入段、停留段和恢复段可以确定性测试。
45. 作为开发者，我想中心点插值和缓动成为实现契约，以便原生窗口移动和缩放表现一致。

## Implementation Decisions

- 这个功能是倒计时完成反馈的展示增强，不是新的计时器模型。
- 只有倒计时自然完成时才触发巨型庆祝。
- 取消计时器不能触发巨型庆祝。
- 只有倒计时完成瞬间宠物可见，才触发巨型庆祝。
- 如果倒计时完成时宠物隐藏，Deskmon 只发送现有 macOS 通知。
- 只有倒计时完成瞬间宠物没有被拖拽，才触发巨型庆祝。
- 如果倒计时完成时宠物正在被拖拽，Deskmon 只发送现有 macOS 通知。
- 拖拽结束后不补触发延迟的巨型庆祝。
- 巨型庆祝总时长固定为 7 秒。
- 这 7 秒是“巨型庆祝动效”，由进入段、停留段和恢复段组成。
- 进入段为 0s 到 3s。
- 停留段为 3s 到 6s。
- 恢复段为 6s 到 7s。
- 第一版不新增用户可见设置项。
- 巨型庆祝三个阶段都使用现有 `celebrate` 动画形态。
- 巨型庆祝期间宠物窗口内不绘制文字。
- macOS 通知继续作为文字完成提示。
- 巨型庆祝通过临时调整宠物窗口尺寸实现，不创建永久全屏 overlay。
- 常规倒计时完成不能使用全屏透明 overlay。
- 临时巨型窗口保持和普通宠物窗口一致的透明、无边框形态。
- 巨型庆祝期间临时置顶。
- 巨型庆祝不能主动调用窗口聚焦，也不能抢走当前应用焦点。
- 提醒出现在倒计时完成时宠物所在的显示器。
- 如果宠物当前点无法匹配显示器，可以回退到主显示器或第一个可用显示器。
- 巨型目标尺寸基于当前显示器可用工作区计算。
- 巨型窗口保持正方形。
- 目标边长约为当前工作区限制维度的 45%。
- 目标边长需要限制在合理范围内，意图下限约 320px、上限约 560px。
- 最终目标尺寸必须仍然能放进当前可用工作区。
- 目标位置需要感知尺寸：较小巨型提醒可以稍微偏下，较大提醒应更接近中心，避免裁切或过度遮挡屏幕下方。
- 目标位置必须 clamp 到当前显示器可用工作区内。
- 进入段同时插值尺寸和位置。
- 进入段从宠物当前中心点插值到巨型提醒目标中心点。
- 恢复段从巨型提醒中心点插值回原宠物中心点。
- 进入段和恢复段都使用直线路径。
- 进入段使用 ease-out。
- 恢复段使用 ease-in-out。
- 动效不使用弧线路径、绕路、弹跳、弹簧或物理模拟。
- 进入巨型庆祝前，Deskmon 捕获普通宠物中心点和恢复所需的展示状态。
- 如果偏好和屏幕状态没有变化，恢复时回到倒计时完成瞬间捕获的中心点。
- 如果动效期间宠物尺寸设置变化，恢复时使用最新用户选择的尺寸。
- 如果动效期间置顶设置变化，恢复时使用最新置顶设置。
- 恢复时最终宠物位置需要 clamp 到当前可见工作区内。
- 巨型庆祝的临时尺寸和位置变化不能持久化到用户设置。
- 巨型庆祝的临时位置不能覆盖已保存的最后宠物位置。
- 巨型庆祝期间禁用指针拖拽。
- 巨型庆祝期间禁用宠物右键菜单。
- 巨型庆祝期间左键仍然无操作。
- 如果用户在巨型庆祝期间隐藏宠物，动效立即结束。
- 巨型庆祝期间执行隐藏时，应先恢复普通展示状态，再隐藏宠物。
- 如果巨型庆祝期间启动新计时器，庆祝立即结束，宠物回到普通计时等待状态。
- 如果巨型庆祝期间设置变化，恢复时优先使用最新保存的普通尺寸和置顶偏好。
- 如果巨型庆祝期间显示器或工作区变化，动效不需要中途重算目标。
- 如果巨型庆祝期间显示器或工作区变化，最终恢复必须重新 clamp 到当前可见工作区。
- 前端状态应把巨型庆祝建模为独立的临时提醒模式，不要只复用普通 `celebrate` mood。
- 原生窗口层应暴露进入临时庆祝展示和恢复普通展示的小接口。
- 状态策略应该可以在不实际调整 macOS 窗口的情况下测试。

建议拆分的深模块：

- 倒计时完成庆祝策略：判断完成计时应该触发巨型庆祝、只通知、立即恢复，还是被新计时器打断。
- 巨型庆祝动效时间轴：负责进入段、停留段、恢复段的阶段计算、中心点插值、缓动和阶段切换。
- 原生宠物展示控制器：负责临时窗口尺寸、位置、置顶、当前显示器可用工作区定位和恢复。
- 宠物交互保护：负责在临时巨型庆祝期间禁用拖拽和宠物菜单。
- 计时器事件协调：把现有 timer-finished 和 timer-changed 事件接到庆祝策略，不改变单计时器规则。
- Smoke test 清单更新：记录 macOS 特有的位置、焦点、可见性和多显示器行为。

## Testing Decisions

- 测试应验证外部行为和模块契约，不测试私有实现细节。
- 庆祝策略测试应覆盖宠物可见且倒计时自然完成时触发巨型庆祝。
- 庆祝策略测试应覆盖宠物隐藏且倒计时自然完成时只发送通知。
- 庆祝策略测试应覆盖宠物拖拽中且倒计时自然完成时只发送通知。
- 庆祝策略测试应覆盖取消计时器不会触发巨型庆祝。
- 庆祝策略测试应覆盖庆祝期间启动新计时器会结束庆祝并进入计时等待。
- 庆祝策略测试应覆盖 7 秒结束后恢复普通状态。
- 动效时间轴测试应覆盖 0s 到 3s 进入段、3s 到 6s 停留段、6s 到 7s 恢复段。
- 动效时间轴测试应覆盖所有阶段都请求 `celebrate` 形态。
- 动效时间轴测试应覆盖进入段和恢复段同时插值尺寸和位置。
- 动效时间轴测试应覆盖插值使用中心点而不是左上角。
- 动效时间轴测试应覆盖进入段使用 ease-out、恢复段使用 ease-in-out。
- 动效时间轴测试应覆盖停留段保持巨型尺寸和目标位置稳定。
- 展示计算测试应覆盖小、中、大工作区下的巨型目标尺寸。
- 展示计算测试应覆盖最小和最大巨型尺寸限制。
- 展示计算测试应覆盖最终尺寸不会超过当前工作区。
- 展示计算测试应覆盖尺寸感知的纵向定位和工作区 clamp。
- 展示恢复测试应覆盖偏好不变时恢复捕获的中心点、尺寸和置顶状态。
- 展示恢复测试应覆盖恢复时尊重变化后的宠物尺寸偏好。
- 展示恢复测试应覆盖恢复时尊重变化后的置顶偏好。
- 展示恢复测试应覆盖显示器或工作区变化时最终通过 clamp 保证可见。
- 持久化测试应覆盖临时巨型尺寸和位置不会覆盖已保存偏好或最后位置。
- 交互测试应覆盖巨型庆祝期间拖拽开始被忽略。
- 交互测试应覆盖巨型庆祝期间右键菜单被忽略。
- 交互测试应覆盖动效期间执行隐藏会恢复普通展示并隐藏宠物。
- 既有计时器测试应继续覆盖单计时器、取消、完成和菜单状态。
- Smoke test 应验证巨型提醒出现在宠物当前所在显示器。
- Smoke test 应验证巨型提醒显示在普通窗口上方且不抢键盘焦点。
- Smoke test 应验证宠物隐藏时倒计时完成只显示 macOS 通知。
- Smoke test 应验证宠物拖拽中倒计时完成不会调整窗口尺寸。
- Smoke test 应验证宠物前 3 秒变大并移动、3s 到 6s 停留、6s 到 7s 恢复。
- Smoke test 应验证宠物 7 秒后恢复。
- Smoke test 应验证巨型庆祝期间启动新计时器会恢复宠物并进入等待状态。

## Out of Scope

- 全屏透明 overlay。
- 整屏接管动画。
- 点击穿透。
- 音效。
- 用户可配置提醒时长。
- 用户可配置提醒尺寸。
- 巨型庆祝设置开关。
- 桌面文字气泡或宠物窗口内的“计时结束啦”等文字。
- 多个计时器。
- 计时历史。
- 稍后提醒、重复提醒或闹钟流程。
- 宠物窗口外的桌面粒子动画。
- 全局键盘快捷键。
- 倒计时完成时抢焦点或激活应用。
- 改变宠物隐藏状态下的倒计时完成行为。
- 拖拽结束后补触发延迟庆祝。
- 巨型庆祝动效中的弧线路径、弹跳、弹簧或物理模拟。
- 瞬间变大或瞬间恢复。

## Further Notes

- 这个 PRD 是对现有桌宠 V1 倒计时完成反馈的聚焦增强。
- 核心产品决策是“更显眼，但仍然礼貌”：足够大、短暂、不可交互、不抢焦点。
- 隐藏状态和主动拖拽都代表用户正在控制宠物；倒计时完成不能覆盖这些控制状态。
- 实现应保留现有规则：macOS 通知是可靠的倒计时完成兜底提示。
- 本 PRD 的规范术语是：“巨型庆祝提醒”表示产品行为，“巨型庆祝动效”表示 7 秒进入、停留、恢复动画。
