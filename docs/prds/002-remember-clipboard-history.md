# PRD：Deskmon 记忆力剪贴板历史

状态：需求已通过 grill-me 收敛，待实现拆分
目标 triage 标签：ready-for-agent
创建日期：2026-06-30
最后更新：2026-07-03

## 问题陈述

用户在工作中会连续复制多段文本，但系统剪贴板通常只保留最后一次复制的内容。用户希望 Deskmon 能像小宠物一样帮忙“记住”这些复制过的文本：复制了几次之后，可以快速回溯并重新拿回其中某一次复制的内容。

这个功能不应该默认把所有剪贴板内容永久保存。剪贴板里可能出现隐私内容、验证码、密码、日志或大段临时文本，所以 V1 必须把“临时捕获”和“主动记住”分开：自动捕获只保存在运行期内存中，用户明确点“记住”后才加密持久化。

## 解决方案

新增一个名为“记忆力”的剪贴板历史功能。Deskmon 运行后自动捕获纯文本剪贴板变化，并把最近复制的文本放入运行期临时历史“记忆中”。用户可以从菜单栏“记忆力”打开“记忆力”窗口，查看记忆中的内容、把某条内容主动放进“笔记本”、把旧内容通过“回忆”写回系统剪贴板、忘记临时或已保存内容。菜单栏还提供“记住刚想到的”快捷动作，用于直接读取当前系统剪贴板文本并保存到“笔记本”。

宠物右键小菜单中新增轻量入口“回忆”。“回忆”分为“记忆中”和“笔记本”两个子菜单，用户可以直接点击某条文本预览，把对应内容重新写回系统剪贴板。小菜单只负责快速重置剪贴板内容，不负责保存、忘记、放在最上面等管理操作。

V1 只支持纯文本。图片、文件、富文本、自动粘贴、全局快捷键、搜索和导出备份都不在范围内。

9. 作为 macOS 用户，我想点击历史条目后只写回系统剪贴板，以便我自己决定在哪里粘贴。

17. 作为 macOS 用户，我想“记忆中”里的条目可以单条忘记，以便清理不想继续显示的临时内容。
18. 作为 macOS 用户，我想“记忆中”只支持单条忘记，以便避免误清空临时历史。
19. 作为 macOS 用户，我想“笔记本”里的条目可以忘记，以便移除不再需要的长期内容。
20. 作为 macOS 用户，我想忘记“笔记本”里的内容时使用系统确认，以便减少误删持久内容的风险。
21. 作为 macOS 用户，我想“笔记本”里的多条内容都可以放在最上面，以便常用文本总是排在前面。
22. 作为 macOS 用户，我想放在最上面和取消只在“记忆力”窗口里完成，以便宠物小菜单保持轻量。
23. 作为 macOS 用户，我想“笔记本”按是否放在最上面和最近保存顺序排序，以便最近或最重要的内容更容易找到。

25. 作为 macOS 用户，我想用旧内容做回忆后，它会移动到“记忆中”第一条，以便最近剪贴板历史符合直觉。

28. 作为 macOS 用户，我想复制内容前后的空白不会制造重复条目，以便历史更干净。

32. 作为 macOS 用户，我想如果加密 key 或数据损坏，应用提示我重置“记忆力”的数据，以便我知道数据不可恢复而不是被静默清空。
33. 作为开发者，我想把剪贴板捕获、历史状态、加密存储、菜单构建和窗口 UI 分成清晰模块，以便分别测试和迭代。
34. 作为开发者，我想加密方案不使用硬编码密钥，以便不同安装不会共用同一把密钥。
35. 作为开发者，我想“记忆中”完全不落盘，以便临时历史的隐私边界简单可靠。

## 实现决策

- 功能名和窗口名使用“记忆力”。
- 菜单栏新增“记忆力”入口，打开独立“记忆力”窗口。
- 菜单栏“记忆力”和“记住刚想到的”作为一级入口同组展示，前后用分割线和其他菜单项隔开；顺序为“记忆力”在上，“记住刚想到的”在下。
- “记住刚想到的”读取当前系统剪贴板纯文本，执行标准化、截断和笔记本保存；它不主动写入“记忆中”，避免和 500ms 剪贴板轮询产生竞态或顺序抖动。
- “记住刚想到的”成功和失败都使用 macOS 系统通知反馈：普通成功为“Deskmon 记住刚想到的了”，截断成功为“Deskmon 记住了前 5000 个字”，空/非文本/空白为“Deskmon 没看到能记住的文字”，笔记本满为“笔记本满啦，先忘记一些再记”，数据或密钥问题为“笔记本打不开了，请到记忆力里重置”。
- 宠物右键小菜单新增“回忆”入口。
- “回忆”下分两个子菜单：“记忆中”和“笔记本”。
- 宠物右键小菜单只负责重置剪贴板内容，不提供保存、忘记、放在最上面、清空、搜索或设置。
- V1 UI 文案采用轻量宠物语气，但必须保持可理解：功能、窗口、菜单栏入口叫“记忆力”，临时历史叫“记忆中”，持久历史叫“笔记本”，宠物小菜单入口和取回动作叫“回忆”，保存动作叫“记住它”，删除动作叫“忘记”，排到前面的动作叫“放在最上面”。
- V1 只处理纯文本剪贴板内容。
- 自动捕获剪贴板变化，不要求用户手动保存当前剪贴板才进入临时历史。
- 剪贴板捕获从应用启动后立即开始，但启动时已有的剪贴板内容不加入“记忆中”。
- 剪贴板捕获采用 500ms 轮询。
- “记忆中”只保存在运行期内存中，退出应用后清空。
- “记忆中”最多 10 条。
- “笔记本”由用户主动点击“记住它”生成。
- “笔记本”持久化保存，最多 50 条。
- “笔记本”满 50 条时，保存新内容失败并提示用户先忘记一些旧内容；不会自动覆盖最旧条目。
- 同样内容重复进入“记忆中”时，不新增重复项，而是移动到第一条。
- 同样内容重复保存为“笔记本”时，不新增重复项，而是刷新内部顺序，并保留原有放在最上面状态。
- 重复判断先对内容执行首尾 trim；中间空格、换行和格式保留。
- 单条文本最多 5,000 字符。
- 超过 5,000 字符的文本会截断到 5,000 字符，并记录截断标记。
- 小菜单条目预览截断到 20 个字符。
- 小菜单“回忆”里的条目不显示序号，只显示文本预览；放在最上面的笔记本条目可保留轻量标识。
- 预览不单独落盘；打开窗口或菜单时从运行时文本生成。
- “记忆力”窗口采用列表预览加详情区完整内容的结构。
- “记忆力”窗口默认内尺寸使用 920x620，最小内尺寸不低于 780x520，保持小宠物应用的轻量感。
- “记忆中”为空时只保留短提示块，不应占掉半个列表区；“笔记本”需要获得主要列表空间。
- “记忆力”窗口不提供手动刷新按钮，打开后依靠事件实时刷新。
- “记忆力”窗口左侧列表承载回忆、记住它、忘记、放在最上面等操作；操作按钮在条目 hover 或键盘 focus 时显示。
- “记忆力”窗口右侧详情区只用于确认完整文本和状态标识，不放底部操作按钮。
- “记忆力”窗口中，“记忆中”条目支持回忆、记住它、忘记单条。
- “记忆力”窗口中，“笔记本”条目支持回忆、放在最上面、取消放在最上面、忘记。
- “笔记本”支持多条内容同时放在最上面；置顶不是单选，也不会自动取消其他条目的置顶状态。
- “笔记本”不支持编辑内容。
- “笔记本”执行忘记时需要使用系统原生确认交互。
- “笔记本”忘记确认弹窗关闭后，无论用户确认还是取消，焦点都应回到“记忆力”窗口。
- “记忆中”忘记不需要二次确认。
- “笔记本”排序规则为：放在最上面的内容在上，内部按放在最上面的顺序倒序；其他内容按最近保存顺序倒序。
- 取消放在最上面后，该条内容会排到非置顶内容的第一条，而不是回到原来的旧位置。
- 从历史中执行回忆时，只写回系统剪贴板，不自动粘贴到当前应用。
- 从历史中执行回忆后，该内容会进入“记忆中”第一条，并按重复规则移动而不是新增重复。
- “记忆力”窗口打开后需要实时刷新记忆中和笔记本的内容。
- 空状态需要短提示：“记忆中”说明复制文本后会被 Deskmon 临时想到这里且退出后清空；“笔记本”说明点击“记住它”后会加密保存在本机。
- 不做敏感内容判断、密码识别、验证码过滤或按应用黑名单过滤；是否记住交给用户判断。
- 持久化存储不允许任何文本明文落盘。
- 加密方案采用本地随机 key 文件加加密数据文件。
- 随机 key 首次需要时生成，保存在本机应用配置目录中。
- 加密数据文件只保存密文和必要元数据，例如 id、内部排序值、是否放在最上面、截断标记；不记录复制时间或保存时间。
- 该加密方案不是强安全模型；如果攻击者同时拿到 key 文件和加密数据文件，理论上可以解密。它的目标是避免文本明文落盘，并避免所有安装共享硬编码密钥。
- 如果 key 文件或加密数据损坏导致无法解密，“笔记本”不可恢复，应提示用户重置“记忆力”的数据。
- 重置“记忆力”的数据需要删除旧 key 和旧加密数据，再重新开始。
- 不做导出、导入或备份。

建议拆分的深模块：

- 剪贴板监视模块：负责轮询系统剪贴板、判断变化、标准化文本和派发捕获事件。
- 记忆力状态模块：负责“记忆中”和“笔记本”的去重、排序、上限、截断、放在最上面和忘记规则。
- 加密存储模块：负责本地随机 key、加密数据读写、解密失败状态和重置。
- 原生菜单模块：负责菜单栏“记忆力”和宠物小菜单“回忆”的结构与事件映射。
- 记忆力窗口模块：负责列表、详情区、实时刷新、空状态和管理操作。
- 系统交互模块：负责写回剪贴板和“忘记”确认等原生交互。

## 测试决策

- 测试应验证外部行为和模块契约，不测试私有实现细节。
- 剪贴板监视测试应覆盖启动时不捕获已有内容、500ms 轮询捕获变化、空文本忽略、首尾 trim、超长截断。
- 记忆力状态测试应覆盖“记忆中”最多 10 条、重复移动到第一条、回忆后回到第一条、忘记单条。
- 记忆力状态测试应覆盖“笔记本”最多 50 条、重复保存刷新内部顺序、满 50 新内容保存失败、多条放在最上面排序、取消放在最上面、忘记。
- 加密存储测试应覆盖文本不明文落盘、重新启动后能读取已保存内容、key 丢失或数据损坏时进入不可恢复错误状态。
- 菜单测试应覆盖宠物小菜单“回忆”同时包含“记忆中”和“笔记本”，两类为空时显示“还没有”，条目预览截断到 20 个字符且不显示序号。
- 菜单测试应覆盖点击历史条目只写回剪贴板，不自动粘贴。
- 窗口测试应覆盖列表预览、行内 hover/focus 操作、详情区完整文本、截断标识、空状态、记住它、放在最上面、取消放在最上面、忘记和实时刷新。
- 删除确认测试应覆盖忘记“笔记本”调用系统确认，取消确认不会删除，确认后才删除。
- 端到端 smoke test 应覆盖连续复制 5 次、从小菜单“回忆”第 3 条、打开“记忆力”窗口记住一条、重启后仍能看到笔记本内容、记忆中重启后清空。

## 不在范围内

- 图片剪贴板历史。
- 文件剪贴板历史。
- 富文本、HTML 或格式化文本恢复。
- 自动粘贴到当前应用。
- 全局快捷键。
- 搜索。
- 编辑已记住内容。
- 云同步。
- 导出、导入或备份。
- 用户密码派生密钥。
- macOS Keychain / Windows Credential Manager / Linux Secret Service 抽象。
- 敏感内容自动识别、密码过滤、验证码过滤或应用黑名单。
- 无限保存数量。
- 强安全承诺或抵抗攻击者同时获取 key 文件和加密数据文件的场景。

## 进一步说明

- 这是 001 桌面像素史莱姆 PRD 之后的独立功能 PRD，不应把 001 的 V1 桌宠边界继续扩大。
- “记忆力”是 Deskmon 从桌面陪伴走向轻量工作辅助的第一项功能，但仍应保持克制。
- “记忆中”是临时历史，“笔记本”是用户主动保存的加密持久历史，这个边界不能混淆。
- 发布状态：当前仓库没有可用的 issue tracker 创建工具；本 PRD 先作为实现基准文档维护。
