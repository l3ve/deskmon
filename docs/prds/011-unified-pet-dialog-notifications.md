# PRD：Deskmon 统一宠物对话提醒与单次倒计时

状态：首轮实现已完成，待 macOS 实机验收
目标 triage 标签：ready-for-agent
创建日期：2026-07-24
最后更新：2026-07-24
发布状态：已发布到 GitHub issue #21：https://github.com/l3ve/deskmon/issues/21

## 问题陈述

Deskmon 当前有多套差异明显的提醒和计时体验。

本地工具通过 `deskmon notify` 发送消息时，宠物会临时移动到鼠标所在屏幕中央、变成醒目尺寸并显示 RPG 对话框。截图结果和“记住刚想到的”结果继续通过 macOS 系统通知传达。专注计时则包含三个快捷时长、休息时长、两类完成文案、休息声音和五阶段连续流程；专注或休息结束后还会进入放大居中的持久交接状态。

这些路径虽然都能完成任务，但用户看到的是几套割裂的反馈。普通桌宠突然离开原位并移动到屏幕中央，会打断当前视线；相同的 Deskmon 事件有时由宠物表达，有时由通知中心表达；专注与休息流程远超用户真正需要的“开启一个倒计时，到点提醒我”。

用户希望 Deskmon 始终保持“小宠物就在桌面原位说话”的感觉。只要宠物可见，外部消息、截图、记忆力和倒计时完成都应使用统一的宠物对话框，不再出现宠物放大、居中和缩回过程。宠物隐藏时仍需要可靠的系统通知兜底，但不能因为提醒自动把宠物叫出来。

用户同时希望彻底移除专注、休息和连续轮次概念，只保留一个可以在设置页配置分钟数的单次倒计时。倒计时只需要启动、显示剩余时间、允许取消和到点提醒，不需要暂停、追加时间、自动重复或完成后的操作选择。

这次调整是提醒呈现系统与倒计时能力的收敛，不是通知中心、消息历史、随机聊天或完整计时器工具。

## 解决方案

Deskmon 建立一套统一的“宠物对话提醒”系统。宠物可见时，所有原本走 macOS 系统通知或中央宠物提醒的事件，都改为在宠物当前位置旁显示 RPG 像素对话框。宠物保持用户选择的常态尺寸，不移动到屏幕中央，不放大，也不执行缩回动画。

统一普通消息来源包括：

- `deskmon notify` 外部提醒。
- 截图保存成功和截图失败。
- “记住刚想到的”成功、截断、空内容、容量和数据异常反馈。
- 单次倒计时自然完成。
- 其他当前通过 macOS 系统通知发送的非交互反馈。

所有普通消息进入一个先进先出队列并逐条播放。消息自动消失，不提供持久操作卡片。倒计时自然完成时，计时状态立即回到未运行，并生成一条普通消息：`XX 分钟倒计时结束啦`。

宠物隐藏时，不显示宠物对话框，也不自动显示宠物。Deskmon 改用 macOS 系统通知发送本次反馈。系统通知送达后不在重新显示宠物时重播。

倒计时只保留两个状态：

```text
未运行 → 运行中 → 未运行
             ↘ 取消 → 未运行
```

设置页只保存一个 `倒计时时长`，默认 30 分钟，范围 1–180 分钟整数。菜单栏和宠物右键菜单在未运行时直接显示 `XX 分钟倒计时`；运行中显示 `倒计时：MM:SS` 和 `取消倒计时`。

Deskmon 应用完全未运行时，`deskmon notify` 继续静默返回成功，不自动启动应用，也不由 CLI 独立发送系统通知。运行中的倒计时不持久化，退出 Deskmon 等同于取消。

## 用户流程

### 宠物可见时的普通消息

1. Deskmon 接收到一个普通消息事件。
2. 如果用户正在拖拽宠物，或区域截图/标注任务正在进行，消息先进入队列但暂不展示。
3. 可展示后，宠物停止自主移动，保持当前位置和常态尺寸。
4. 宠物与对话框临时置顶，但不获取当前应用的键盘焦点。
5. 成功或普通消息让宠物按原尺寸播放一轮 `celebrate`；错误消息保持 `idle`。
6. 当前消息使用逐字动画显示标题和正文。
7. 普通或成功消息显示 4 秒；错误消息显示 8 秒。
8. 当前消息结束后，下一条按到达顺序播放。
9. 队列清空后，对话框收起，宠物恢复原置顶设置和自主活动节奏。

### 启动倒计时

1. 用户从菜单栏或宠物右键菜单点击 `30 分钟倒计时`。
2. Deskmon 以当前设置值锁定本轮时长并开始计时。
3. 不显示“倒计时已开始”对话框。
4. 宠物切换为 `timer-waiting`，仍按活动设置在活动区域内移动并支持拖拽。
5. 倒计时期间暂停鼠标追逐。
6. 两个菜单入口同步切换为只读状态 `倒计时：29:59` 和操作 `取消倒计时`。

### 取消倒计时

1. 用户从任一菜单点击 `取消倒计时`。
2. Deskmon 立即清除本轮结束时间并回到未运行。
3. 取消不播放动画、不发送通知，也不显示取消成功消息。
4. 菜单恢复为当前设置对应的 `XX 分钟倒计时`。
5. 宠物恢复普通活动状态；如果用户原本暂停了移动，继续尊重暂停设置。

### 倒计时自然完成

1. 到达本轮结束时间后，Deskmon 只完成一次。
2. 计时状态立即回到未运行，菜单恢复为 `XX 分钟倒计时`。
3. 生成普通消息 `XX 分钟倒计时结束啦`，其中分钟数使用本轮锁定值。
4. 宠物可见时，消息进入统一先进先出队列，不打断正在展示的更早消息。
5. 宠物隐藏时，Deskmon 发送相同文案的 macOS 系统通知。
6. 不播放独立提示音，不显示操作按钮，不自动开始下一轮。

### 宠物隐藏

1. 宠物隐藏时仍可从菜单栏启动倒计时。
2. 隐藏不会取消已经运行的倒计时。
3. 计时期间宠物保持隐藏，不为展示 `timer-waiting` 自动恢复。
4. 如果用户在到点前重新显示宠物，宠物使用 `timer-waiting`，菜单继续显示剩余时间。
5. 如果倒计时在隐藏期间完成，只发送 macOS 系统通知。
6. 重新显示宠物后不重播已经送达的倒计时消息。

### 设置变更

1. 设置页只负责修改倒计时时长，不提供启动按钮。
2. 倒计时运行中仍允许修改设置。
3. 修改后的时长只用于下一轮，不影响当前结束时间。
4. 设置页不额外显示“下次生效”提示。

### 休眠与退出

1. Mac 进入系统休眠时，倒计时继续按真实时间流逝。
2. 唤醒时如果已经到点，Deskmon 立即完成一次并发送提醒。
3. 不因长时间休眠重复发送完成提醒。
4. 退出 Deskmon 等同于取消，不保存运行中的倒计时。
5. 重新启动后不恢复倒计时，也不补发过期提醒。

### 拖拽

1. 对话框存在时，宠物停止自主移动但仍可被用户拖拽。
2. 用户拖动宠物时，宠物和全部可见对话框作为一个整体移动。
3. 拖拽期间暂停普通消息的自动消失计时。
4. 新消息在拖拽期间正常入队，但等用户松手后才开始展示。
5. 松手后保存宠物的新常态位置，再根据当前屏幕可用空间重新计算对话框方向。
6. 换边只在松手后发生，避免拖动过程中左右跳动。

## 菜单与设置

### 菜单栏与宠物右键菜单

未运行：

```text
30 分钟倒计时
```

运行中：

```text
倒计时：29:59
取消倒计时
```

菜单规则：

- 删除“专注计时”子菜单。
- `XX 分钟倒计时` 是直接操作项，点击即开始。
- 同一时间只允许一个倒计时。
- 运行中不提供新的开始入口。
- `倒计时：MM:SS` 是只读状态项，每秒更新已有菜单项文本。
- 菜单栏与宠物右键菜单必须共享同一计时状态。
- 不提供暂停、继续、增加 5 分钟、重新开始或自动重复。

### 设置页

```text
倒计时
倒计时时长  [ − ] 30 [ + ] 分钟
```

设置规则：

- 面板标题使用 `倒计时`。
- 唯一字段名称使用 `倒计时时长`。
- 默认值为 30 分钟。
- 只接受 1–180 分钟整数。
- 继续复用现有紧凑步进控件，不依赖原生 number spinner。
- 侧栏摘要显示 `30 分钟`。
- 不提供开始倒计时按钮。
- 不提供完成文案设置。
- 不提供休息时长和休息完成文案。
- 不提供完成声音开关。
- 不在运行中显示“修改将在下次生效”说明。

### 旧设置迁移

- 新安装和没有旧计时配置时使用 30 分钟。
- 已有三个专注快捷时长的用户，迁移时取排序后的中间值作为新的倒计时时长。
- 已有用户自定义值不能被默认 30 分钟覆盖。
- 旧休息时长、专注完成文案、休息完成文案和声音开关不再写入新设置结构。
- 读取旧设置完成迁移后，后续保存只写新的单值倒计时配置。
- 无效旧值统一回退到 30 分钟。

## 布局与视觉

统一对话框使用现有 RPG 像素风格：深色半透明面板、浅色系统字体、像素边框、标题名牌和克制阴影。错误消息使用克制的红色像素强调线，不新增错误宠物素材。

```text
                    ┌──────────────────────┐
                    │ 当前普通消息          │
                    │ 后方最多露出三层叠卡  │
         共用尖角 ◀ └──────────────────────┘
             [ 常态尺寸宠物 ]
```

布局规则：

- 宠物位置和尺寸始终作为稳定锚点。
- 对话框优先放在宠物右侧；右侧不足时放左侧；两侧都不足时放到宠物下方。
- 宠物本体仍受用户自定义活动区域约束。
- 对话框不受活动区域限制，只需完整留在当前显示器的可用工作区内，避开菜单栏和 Dock。
- 多显示器场景始终使用宠物当前所在显示器；拖拽跨屏后按松手位置重新选择工作区。
- 入场先使用约 160ms 的像素步进展开和轻微淡入。
- 退场先使用约 100ms 的像素步进收起。
- 不从屏幕边缘飞入，不推动宠物，不改变宠物尺寸。
- 入退场参数属于后续可以继续调校的表现细节，不改变状态与交互契约。

## 普通消息队列

普通消息队列遵守以下规则：

- 严格按到达顺序播放，错误消息不插队。
- 当前正在显示的消息不会被新消息打断。
- 实际队列最多保留 20 条，包括当前消息。
- 超过 20 条时保留当前消息，淘汰最早的待播放消息。
- 视觉叠卡最多显示 4 层；即使还有 5 条以上消息，也只显示 4 层。
- 当剩余消息少于 4 条时，叠卡层数才逐层减少。
- 叠卡不显示 `1 / 3`、“还有 N 条”或可翻页控件，不暗示用户可以返回上一条。
- 相同最终标题和正文在短时间内重复到达时，合并为同一条并显示 `×2`、`×3` 等计数。
- 重复消息合并后重新计算当前消息的停留时间。
- 普通或成功消息从实际出现时开始计时 4 秒。
- 失败、权限或数据异常消息从实际出现时开始计时 8 秒。
- 排队、截图延迟和拖拽延迟期间不提前消耗展示时间。
- 鼠标悬停在当前普通消息上时暂停计时，移开后继续剩余时间。
- 拖拽宠物与对话框组时暂停计时，松手并完成布局后继续。
- 点击仍在逐字显示的消息时，第一次点击只立即补全文案。
- 文案已经完整显示时再次点击，立即结束当前消息并播放下一条。
- 普通消息不提供关闭按钮、返回上一条或清空全部队列的控件。

## 逐字动画

- 普通消息的标题和正文逐字出现。
- 普通消息的展示时长包含逐字动画所占时间。
- 点击未完成文案时只补全当前文字，不跳过当前消息。
- 倒计时菜单文本、设置字段和系统通知不使用逐字动画。

## 焦点、层级与输入

- 消息出现时，宠物窗口临时置顶，但不能主动获取键盘焦点。
- 普通消息队列清空后，恢复用户当前的“默认置顶”设置。
- “默认置顶”设置继续保留，不改为永久强制置顶。
- 点击普通消息是明确的用户操作，可以完成文字或前进一条。
- 透明空白区域保持点击穿透，不能因为窗口扩展阻挡附近应用。
- 宠物本体继续支持拖拽和右键菜单。
- 启动或取消倒计时不主动切换桌面空间或聚焦宠物窗口。

## 通知路由

| 条件 | 普通消息与倒计时完成 |
| --- | --- |
| 宠物可见且空闲 | 加入对话队列并原地展示 |
| 宠物可见且正在拖拽 | 入队，松手后展示 |
| 宠物可见且截图/标注中 | 入队，任务退出后展示 |
| 宠物隐藏 | 发送 macOS 系统通知，不重播 |
| Deskmon 未运行 | `deskmon notify` 静默成功；倒计时不存在 |

系统原生确认框、目录选择器和屏幕录制权限引导不属于通知路由，继续保留现有系统交互。

统一普通消息不新增独立声音。倒计时完成不播放专用声音；宠物隐藏时的系统通知按系统通知行为处理。

## 实现决策

- 新建统一消息契约，至少包含消息标识、来源、标题、正文、普通/成功/错误语义、重复计数、接收时间和展示时长。
- 所有普通通知来源先映射到统一消息契约，再交给同一队列；通知来源不直接控制窗口位置和动画。
- 普通消息状态模块负责规范化、重复合并、先进先出、20 条容量、当前消息、计时、悬停暂停、拖拽暂停、截图暂停和点击前进。
- 单次倒计时状态模块只公开开始、取消、完成和快照能力；内部状态只能是未运行或运行中。
- 运行中快照至少包含本轮分钟数、持续秒数、结束时间和剩余秒数。
- 开始时复制当前设置分钟数，本轮结束时间不再依赖后续设置变更。
- 完成动作必须校验活动计时标识，避免旧异步 worker 在取消或新一轮开始后误发通知。
- 取消必须使旧计时 worker 失效，并且不能生成普通消息。
- 自然完成先原子地清空运行状态，再发布快照和倒计时完成消息，避免菜单停留在运行中。
- 倒计时完成通过统一消息契约进入普通消息队列，不建立专用完成卡片。
- 倒计时可见性路由与其他普通消息一致；隐藏时发系统通知，可见时只发宠物对话。
- 设置模块使用单个倒计时分钟值，默认 30，并集中执行 1–180 校验和旧结构迁移。
- 旧设置迁移读取排序后的三个快捷时长中间值；迁移后的保存结果不再包含旧休息、文案和声音字段。
- 菜单映射模块只处理未运行和运行中：未运行映射直接开始项，运行中映射剩余时间和取消项。
- 两个菜单入口复用同一映射结果和倒计时快照，不维护独立状态。
- 前端宠物只根据 `isRunning` 映射 `timer-waiting` 和鼠标追逐阻塞，不处理休息或待选择阶段。
- 删除专注/休息五阶段状态、动作枚举、流程卡片、中央交接和独立完成声音的产品依赖。
- 呈现模块以宠物当前位置、宠物尺寸、显示器可用工作区和消息卡片尺寸为输入，输出窗口边界、宠物偏移、卡片偏移与右/左/下方向。
- 窗口扩展必须保持宠物的屏幕坐标稳定；不能因窗口左上角变化让宠物视觉上发生位移。
- 拖拽命中以宠物本体为入口，移动的是宠物与对话框组；松手后把宠物锚点换算为新的常态位置并持久化。
- 标题和正文逐字状态由统一文本展示模块管理。
- 宠物可见时不发送对应的 macOS 系统通知，避免宠物对话和通知中心重复提醒。
- 宠物隐藏时由后端通知适配器发送系统通知；消息发送后不在前端重播。
- 系统通知权限被拒绝时，不自动显示隐藏宠物。
- 区域截图和标注期间冻结新对话框展示，但不冻结消息入队或倒计时完成。
- 原生确认、目录选择和权限引导继续使用现有系统对话框，不进入普通消息队列。
- 外部 CLI 的参数、标题正文限制、Socket 边界、安装与卸载规则保持不变。
- `deskmon notify` 在应用未运行时继续静默成功；不新增后台守护进程或 CLI 系统通知适配器。
- 保留“默认置顶”设置；对话呈现期间临时置顶，退出后读取并恢复最新偏好。
- 消息出现时不主动聚焦；只有用户点击后才进入对应交互。
- 不再调用宠物放大、中央移动和缩回流程来呈现通知或倒计时完成。
- 保留原尺寸 `timer-waiting` 和 `celebrate` 动画素材，不新增倒计时专用 sprite。
- 原巨型庆祝、轻量专注计时和外部提醒中央呈现的历史 PRD 不删除；本 PRD 对冲突的产品行为具有更高优先级。

## 测试决策

测试只验证用户可见行为和稳定接口，不依赖 DOM 内部结构、具体 CSS 属性或动画逐帧实现。

### 单次倒计时状态

- 覆盖未运行时可以开始，运行中不能再次开始。
- 覆盖开始时锁定分钟数和结束时间。
- 覆盖剩余秒数按当前时间计算并向上取整。
- 覆盖取消回到未运行且不产生完成消息。
- 覆盖自然完成回到未运行且只产生一次完成消息。
- 覆盖取消后旧 worker 到达时不会误完成。
- 覆盖上一轮 worker 不会完成新一轮倒计时。
- 覆盖休眠或长时间跳过结束时间后立即完成一次。
- 覆盖运行中设置变化不改变当前结束时间。
- 覆盖运行状态不跨应用重启恢复。

### 设置与迁移

- 覆盖新安装默认 30 分钟。
- 覆盖 1 分钟和 180 分钟边界。
- 覆盖 0、负数、非整数和超过 180 的拒绝或规范化。
- 覆盖旧三个快捷时长迁移到排序后的中间值。
- 覆盖旧值无效时回退到 30 分钟。
- 覆盖迁移后保存结果不再包含休息、完成文案和声音字段。
- 覆盖已有用户值不会被默认值覆盖。

### 菜单映射

- 覆盖未运行时两个入口都显示 `XX 分钟倒计时`。
- 覆盖运行中两个入口都显示 `倒计时：MM:SS` 和 `取消倒计时`。
- 覆盖取消后两个入口同步恢复。
- 覆盖自然完成后两个入口同步恢复。
- 覆盖每秒只更新已有状态项，不重建整棵菜单。
- 覆盖宠物隐藏时菜单栏仍能开始和取消。

### 普通消息状态

- 覆盖严格先进先出，错误消息不插队。
- 覆盖当前消息不被新消息打断。
- 覆盖 20 条容量和超过上限时淘汰最早待播消息。
- 覆盖视觉叠卡层数最多为 4，并随剩余数量减少。
- 覆盖重复内容合并、计数更新和重新计时。
- 覆盖普通/成功 4 秒、错误 8 秒。
- 覆盖排队期间不提前消耗展示时长。
- 覆盖悬停、拖拽和截图暂停后按剩余时间恢复。
- 覆盖首次点击补全文案、再次点击前进。

### 统一通知路由

- 覆盖宠物可见时只发宠物对话，不发系统通知。
- 覆盖宠物隐藏时只发系统通知，不自动显示宠物。
- 覆盖隐藏消息不在重新显示后重播。
- 覆盖倒计时完成文案使用本轮分钟数。
- 覆盖倒计时完成与截图、记忆力和外部消息共享 FIFO。
- 覆盖应用未运行时 CLI 静默成功。
- 覆盖截图成功、截图失败和记忆力反馈映射到正确标题、正文与语义。
- 覆盖原生确认和权限引导不进入普通消息队列。

### 原地布局与拖拽

- 覆盖右侧、左侧和下方布局选择。
- 覆盖宠物屏幕坐标在窗口扩展和收起前后保持不变。
- 覆盖活动区域只约束宠物、可用工作区约束完整对话框。
- 覆盖多显示器和不同缩放比例。
- 覆盖拖动中不换边、松手后重新布局。
- 覆盖拖拽后宠物常态位置正确持久化。

### macOS 手工验收

- 验证通知出现和消失时当前应用不丢失键盘焦点。
- 验证临时置顶结束后恢复用户设置。
- 验证透明空白区域点击穿透、卡片点击可用、宠物拖拽可用。
- 验证拖拽跨显示器后的方向、缩放和工作区边界。
- 验证 Dock 位于不同边缘时对话框不被遮挡。
- 验证系统通知权限允许和拒绝两种隐藏宠物路径。
- 验证隐藏宠物时可以从菜单栏启动倒计时并在到点后收到系统通知。
- 验证系统休眠跨过结束时间后只提醒一次。
- 验证截图与标注期间不会出现新对话框，任务退出后立即恢复。
- 验证 160ms/100ms 初始动效不会造成宠物位置跳动。

## 不在范围内

- 专注、休息、连续轮次、自动番茄循环或完成后的操作选择。
- 多个快捷时长、多个并行倒计时或倒计时模板。
- 暂停、继续、追加时间、减少时间或自动重复。
- 倒计时历史、统计、任务名称、标签或完成记录。
- 自定义倒计时完成文案、独立声音、铃声选择或语音播报。
- 倒计时跨应用重启恢复、后台守护进程或登录项。
- 设置页启动倒计时或运行中“下次生效”提示。
- 随机台词、主动闲聊、AI 对话或基于时间的自动气泡。
- 通知历史、已读状态、搜索、归档、跨重启恢复或通知中心窗口。
- 返回上一条消息、手动翻页或清空全部队列控件。
- 远程通知、网络服务、跨设备同步或推送服务。
- 新的宠物状态、错误动画、成功动画或通知专用 sprite。
- 修改 `deskmon notify` 的命令行参数和安装契约。
- 把原生确认框、文件夹选择器和系统权限引导替换为宠物对话框。
- 新增“使用系统通知/使用宠物对话框”设置开关。
- 删除旧 PRD 或重写已经完成的历史 issue。
- 本轮直接优化最终动画曲线；入退场参数先作为可调基线。

## 补充说明

本 PRD 覆盖以下历史规则中与目标体验冲突的部分：

- 巨型庆祝提醒中的宠物放大、中央移动、停留和缩回。
- 轻量专注计时器中的三个快捷时长、休息时长、完成文案、声音、五阶段连续流程和持久操作卡片。
- 本地宠物提醒 CLI 中的中央展示、最多三条同时显示、固定 3 秒和点击穿透消息卡片。
- 截图与记忆力功能中“宠物可见时仍发送系统通知”的反馈方式。

未被本 PRD 明确覆盖的业务规则继续有效，包括截图保存契约、记忆力数据边界、CLI 参数与安全边界、宠物活动区域、菜单栏与右键菜单的原生交互方式。
