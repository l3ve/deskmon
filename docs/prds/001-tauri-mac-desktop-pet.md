# PRD：Tauri macOS 桌面像素史莱姆

状态：V1 范围已通过 grill-me 收敛，待实现拆分
目标 triage 标签：ready-for-agent
创建日期：2026-06-25
最后更新：2026-06-25

## 问题陈述

用户想用 Tauri 做一个 macOS 桌面端小宠物。它不应该像普通应用窗口一样待在固定框里，而是像真的存在于桌面上的小角色：可以在桌面安全区域内活动、跑动、休息、被拖拽，并用轻量方式帮用户完成一个小任务。

V1 的目标不是做完整工作助手，也不是做养成游戏。V1 只需要证明“桌面宠物成立”：它要看起来像一个活着的像素风史莱姆，稳定地出现在 macOS 桌面上，不抢焦点，不明显打扰用户，能被暂停、隐藏、拖走和退出，并通过一个简单计时器验证它可以参与用户工作流。

当前仓库还是空 git 仓库，所以第一版必须克制。优先做桌面宠物的窗口、移动、动画、设置和计时器闭环，不在第一版引入 AI 对话、养成系统、复杂自动化、拖图换皮肤、全屏 overlay 或多动作平台。

## 解决方案

构建一个基于 Tauri 的 macOS 桌面小宠物应用。V1 的主角是一个默认像素风史莱姆，运行在一个小尺寸透明无边框窗口中。窗口本身跟随史莱姆移动，不使用常驻全屏透明 overlay。

V1 包含三层核心能力：

- 宠物窗口：透明、无边框、小尺寸、默认置顶、可拖拽、可隐藏、可从菜单栏恢复。
- 行为与动画：轻量状态机加插值移动，驱动史莱姆在用户活动区域内 idle、walk、run、sleep、timer-waiting、celebrate、dragged。
- 计时器：通过菜单栏或宠物小菜单启动 1 / 5 / 10 / 25 分钟计时；计时中可查看剩余时间并取消；完成时宠物庆祝并发送 macOS 通知。

V1 的产品优先级是桌面陪伴优先，工作助手其次。计时器只是最小动作能力验证点，不把第一版扩展成完整任务管理工具。

## V1 范围

V1 必须实现：

- macOS Tauri 应用脚手架。
- 小尺寸透明无边框宠物窗口。
- 默认像素风史莱姆，先用代码绘制像素风动画，预留未来 sprite sheet / 皮肤接口。
- 默认置顶开启，用户可关闭。
- 菜单栏图标作为稳定控制入口。
- 宠物本体小菜单作为快捷入口。
- 单击或右键宠物打开小菜单。
- 拖拽宠物移动位置，使用鼠标按下时的相对偏移，不让宠物中心突然吸到鼠标位置。
- 小移动视为点击，大移动视为拖拽。
- 不定义双击行为。
- 暂停移动时停止位置变化，但保留安静动画、点击、拖拽和计时器能力。
- 隐藏宠物时宠物窗口不可见，但菜单栏仍可用。
- 最后宠物位置持久化；重启后若位置可见则恢复，否则回到活动区域内。
- 基于当前可用工作区生成默认活动区域，不能假设 Dock 在底部。
- 设置窗口里支持用户用鼠标框选一份自定义活动区域。
- 自定义活动区域必须有最小尺寸限制，并限制在可见工作区内。
- 只保存一份当前活动区域，不做每个显示器独立配置。
- 多显示器只保证宠物可见和可恢复，不做跨屏漫游。
- 轻量状态机加插值移动，不做物理模拟。
- 边界行为是停下、转身、换目标，不做物理反弹。
- 计时器固定选项为 1 / 5 / 10 / 25 分钟。
- 同一时间只允许一个计时器。
- 计时完成时发送 macOS 通知。
- 宠物可见时，计时完成播放 celebrate 动画。
- 宠物隐藏时，计时完成只发送 macOS 通知，不自动显示宠物。
- 宠物暂停但可见时，计时完成原地庆祝。

V1 明确不做：

- 点击穿透。V1 依靠小窗口、安全活动区域、可拖拽、暂停、隐藏和退出控制打扰程度。
- 常驻全屏 overlay。
- 路径特效、桌面粒子、屏幕边缘复杂互动。
- 拖图片给宠物并生成皮肤。
- 逐帧 sprite sheet 美术管线。
- 自定义计时分钟。
- 多个计时器。
- 计时历史。
- 宠物旁边的桌面剩余时间气泡。
- 菜单栏图标旁常驻倒计时文本。
- 快速笔记、打开 URL 或打开应用快捷入口。
- 开机自启动。
- 宠物动作音效。
- AI 对话、聊天、语音、联网模型调用、长期记忆。
- 情绪、亲密度、喂养等养成系统。

## 菜单与交互规格

菜单栏未计时时：

- 显示/隐藏宠物
- 暂停/恢复移动
- 计时器 >
  - 1 分钟
  - 5 分钟
  - 10 分钟
  - 25 分钟
- 设置
- 退出

菜单栏计时中：

- 显示/隐藏宠物
- 暂停/恢复移动
- 计时中：还剩 MM:SS
- 取消当前计时
- 设置
- 退出

宠物小菜单未计时时：

- 计时器 >
  - 1 分钟
  - 5 分钟
  - 10 分钟
  - 25 分钟
- 暂停/恢复移动
- 隐藏

宠物小菜单计时中：

- 计时中：还剩 MM:SS
- 取消当前计时
- 暂停/恢复移动
- 隐藏

宠物小菜单不包含设置，不包含退出。设置和退出只从菜单栏进入。

## 实现决策

- 应用基于 Tauri 构建，V1 目标平台只做 macOS。
- 主宠物界面是一个小尺寸透明无边框窗口，不使用常驻全屏 overlay。
- 窗口尺寸应尽量贴近史莱姆本体，减少遮挡；V1 不做点击穿透。
- 默认置顶开启，但用户可以在设置中关闭。
- 菜单栏图标是稳定控制入口，即使宠物隐藏也必须可用。
- 宠物小菜单只承载即时操作，管理型操作放在菜单栏。
- 默认活动区域根据当前可用工作区推导，不能假设 Dock 在底部。
- 设置窗口提供活动区域框选流程。框选层只在设置流程中短暂出现，不作为日常运行 overlay。
- 活动区域需要最小尺寸限制：宽度至少容纳宠物宽度的 3 倍，高度至少容纳宠物高度的 2 倍。
- 用户拖拽可临时把宠物放到活动区域外，但必须仍在可见工作区内。
- 自主移动恢复后，如果宠物在活动区域外，应以自然方式返回活动区域。
- 只保存一份当前活动区域。显示器不存在或区域不可见时，回退到主屏默认活动区域。
- 多显示器只做可见性和恢复，不做跨屏漫游、每屏配置或多宠物。
- 移动系统使用轻量状态机加插值，不做重力、碰撞、弹跳、避障或路径寻路。
- 边界行为使用停下、转身、换目标，不做反弹。
- V1 动画状态为 idle、walk、run、sleep、timer-waiting、celebrate、dragged。
- V1 使用代码绘制像素风史莱姆动画，放大时保持硬边，不做平滑模糊。
- 宠物大小为小 / 中 / 大三档，中号默认。
- 活跃程度为安静 / 标准 / 活泼三档，不单独暴露移动速度。
- 暂停移动只停止位置变化，不停止动画、点击、拖拽和计时器。
- 隐藏宠物只隐藏窗口，不停止计时器。
- 计时器只支持固定时长 1 / 5 / 10 / 25 分钟。
- 同一时间只允许一个计时器。
- 计时中不保留计时器子菜单，只显示剩余时间和取消入口。
- 剩余时间只在展开菜单后显示，不显示在菜单栏图标旁，也不显示在宠物旁边。
- 计时完成时，宠物可见则播放 celebrate；宠物隐藏则只发 macOS 通知。
- V1 不做开机自启动。
- V1 不做宠物动作音效。
- V1 不做 AI、养成、语音、联网模型调用或长期记忆。
- V1 暂不实现拖图换皮肤，但资产接口要避免把默认史莱姆硬编码到不可替换。

## 测试决策

- 测试应验证外部行为和模块契约，不测试私有实现细节。
- 行为状态机需要确定性单元测试，使用固定时间步长、固定活动区域和可播种随机源。
- 移动测试应覆盖宠物始终保持在活动区域内、自主移动从区域外回归、边界停下转身、暂停后位置不变。
- 拖拽测试应覆盖相对偏移、点击阈值、拖拽后保存最后位置、拖拽到活动区域外但仍在可见工作区内。
- 活动区域测试应覆盖默认区域生成、Dock 不同位置、最小尺寸限制、重置区域、区域不可见时回退。
- 多显示器测试应覆盖显示器移除后宠物回到可见区域。
- 设置测试应覆盖小 / 中 / 大尺寸、安静 / 标准 / 活泼、置顶开关、活动区域持久化。
- 计时器测试应覆盖 1 / 5 / 10 / 25 分钟启动、同一时间只允许一个计时器、取消、完成、隐藏状态完成、暂停状态完成。
- 菜单测试应覆盖未计时和计时中两种菜单结构，确认计时中不显示计时器子菜单。
- 通知测试应验证计时完成时产生 macOS 通知请求，但不直接测试系统通知中心本身。
- 渲染测试应确认七个动画状态都能映射到可渲染像素史莱姆状态。
- 端到端 smoke test 应覆盖应用启动、宠物显示、菜单栏菜单、宠物小菜单、启动 1 分钟计时、取消计时、打开设置、框选活动区域、隐藏与恢复、退出。
- 性能 QA 应检查空闲 CPU、内存稳定性、动画流畅度，以及长时间运行时的电池影响。

- 情绪、亲密度、喂养等养成系统。

## 后续扩展

- V1.1 可以考虑拖入图片生成像素风史莱姆皮肤。推荐语义是仍保持史莱姆造型，但从图片提取颜色、纹理、脸部或装饰元素，而不是把任意图片直接变成完整动画宠物。
- V1.1 可以考虑 sprite sheet / 皮肤资源导入，但需要先定义资产规格和状态映射。
- 如果用户后续明确需要，才重新评估快速提醒、快速笔记、打开 URL/应用、音效、开机启动或更复杂助手能力。
