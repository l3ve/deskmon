# PRD：Deskmon 区域截图原地标注 V1

状态：待实现
目标 triage 标签：ready-for-agent
创建日期：2026-07-15
最后更新：2026-07-15
发布状态：已发布到 GitHub issue #15：https://github.com/l3ve/deskmon/issues/15

## 问题陈述

Deskmon 已经能够从菜单栏和宠物右键菜单发起区域截图，并把 PNG 保存到用户指定目录。但当前流程在框选完成后立即落盘，用户无法在保存前指出问题位置、标记交互路径或补充上下文。

用户使用截图的核心目的之一，是把当前桌面情况交给 AI 分析。单纯截图虽然保留了画面，却经常缺少“看这里”“从这里到这里”“这个区域没有响应”等关注信息。用户需要矩形、箭头和文字三种轻量标注能力，但不希望框选后跳入一个新的普通编辑窗口，让截图和标注变成两段割裂的操作。

本需求将截图升级为一段连续流程：用户在桌面上框选，松开鼠标后选区在原位置冻结，工具栏直接出现在选区旁，用户就地完成标注并保存。整个过程中不隐藏、不暂停、不移动 Deskmon，也不让用户感知到独立编辑窗口。

本 PRD 覆盖原区域截图 PRD 中“框选后立即保存”和“固定使用 macOS 原生框选器”的规则，也覆盖本 PRD 早期版本中的“打开普通、可缩放编辑窗口”方案。保存目录、PNG 命名、权限处理、任务互斥和不干扰桌宠状态等基础能力继续沿用。

## 解决方案

### 一体式流程

1. 用户从现有一级“区域截图”入口发起截图。
2. Deskmon 在所有显示器上建立透明无边框框选层，桌面覆盖一层轻度半透明暗色遮罩。
3. 框选阶段保持系统原有鼠标指针样式，不切换为十字光标。
4. 用户按下并拖动鼠标。选区保持正常亮度，外部继续轻度变暗；选区显示细白色边框和实时尺寸，例如 `820 × 460`。
5. 选区不能跨显示器。按下鼠标所在的显示器决定本次选区归属，拖动到其他显示器时坐标被限制在起始显示器内。
6. 用户松开鼠标时，以该瞬间的桌面内容生成截图，不包含鼠标指针，也不包含遮罩、选区边框和任何截图工具 UI。
7. 如果选区宽或高小于 `10 px`，视为误操作，不进入编辑状态，继续保留框选层供用户重新拖动。
8. 有效选区在原位置冻结，选区保持正常亮度，选区外继续显示轻度暗色遮罩并保持实时变化。
9. 工具栏直接出现在选区旁，用户在冻结画面上原地标注，不打开普通编辑窗口。
10. 用户点击“保存”后才生成最终 PNG；点击 `×` 则直接丢弃本次截图。

### 编辑状态

选区进入编辑状态后保持固定位置和尺寸，不提供移动、缩放或重新裁剪控制。选错区域时，用户点击 `×` 丢弃，再重新发起截图。

冻结选区以外的桌面继续实时变化，并保持与框选阶段一致的轻度暗色遮罩；选区保持正常亮度，形成明确的编辑焦点。多显示器场景下，非选区所在显示器保持整屏遮罩。编辑期间所有选区外点击都由截图层拦截，不能误操作底层窗口。保存成功或丢弃后，截图层完全退出，遮罩消失且桌面交互立即恢复。

工具栏默认位于选区下方居中；下方空间不足时放到选区上方，并始终限制在起始显示器可见范围内。如果选区几乎占满显示器、上下都无法容纳，工具栏覆盖在选区底部内侧。工具栏、边框、错误提示和其他编辑 UI 都不能进入最终 PNG。

工具栏固定提供：矩形、箭头、文字、撤销、丢弃 `×` 和保存。初次进入编辑状态时不默认选择任何标注工具，用户可以直接保存未标注截图。选择工具后，该工具保持激活，可连续创建同类标注，直到用户切换工具或退出。

### 标注规则

- 矩形通过拖拽创建，使用透明填充的空心轮廓。
- 箭头通过起点到终点的拖拽创建，箭头头部位于松开位置。
- 文字工具通过点击选区位置创建原地输入框，支持多行，最多 200 字。
- 文字输入时点击选区内其他位置或切换工具，会完成非空文字；纯空白内容自动取消。
- 矩形、箭头和文字只能显示在选区内，超出部分必须裁切，不得覆盖外部桌面或工具栏。
- 标注主体统一使用醒目红色，并带细白色外描边；V1 不提供颜色、线宽、字体和字号设置。
- 完成后的标注不支持单独选择、移动、缩放、旋转或删除。

撤销只通过工具栏按钮触发：

1. 如果正在绘制或输入文字，先取消当前未完成操作。
2. 如果没有未完成操作，删除最近完成的一条标注。
3. 如果没有可撤销内容，撤销按钮置灰。

V1 不提供重做。

### 保存与丢弃

用户点击保存后，保存按钮立即进入“保存中”状态并禁用，防止重复提交。截图层保持可见，直到保存结果明确返回。

保存成功后，Deskmon 关闭整个截图层、清理临时数据、释放任务锁，并发送无声音的 macOS 系统通知。通知显示“截图已保存”和实际文件名，不自动打开 Finder、图片或其他窗口。截图的框选、编辑和保存全过程都不播放声音。

保存失败时，冻结截图、全部标注和工具状态必须完整保留。错误信息显示在工具栏下方，不弹出新的普通编辑窗口。用户可以修复问题后再次点击保存，也可以点击 `×` 丢弃。

保存目录无效或无法创建时，错误区域提供“重新选择目录”和“恢复桌面”两个动作。只有用户点击“重新选择目录”后才打开系统目录选择框；目录修复完成后不自动重试保存，仍由用户再次点击保存。

用户点击 `×` 时直接丢弃截图、标注和临时数据，不弹确认、不生成文件、不发送通知。框选阶段按 `Escape` 仍然静默取消；进入编辑状态后不提供编辑快捷键，所有工具与命令都通过可见控件操作。

### 生命周期与桌面状态

从开始框选到保存成功、丢弃或框选取消，整段流程视为一个截图任务。同一时间只允许一个任务；任务进行中再次收到“区域截图”请求时静默忽略。

截图与编辑期间，宠物动画、自主移动、设置窗口、“记忆力”、专注计时和中央交接流程继续正常运行。Deskmon 不隐藏、不暂停、不移动这些窗口，也不修改宠物大小、位置、可见性、置顶偏好或专注状态。用户框选到宠物或其他 Deskmon 窗口时，它们应按松开鼠标时的真实状态出现在截图中。

内部可以使用应用缓存保存原始像素和编辑会话，但缓存不是用户文件，不能写入最终保存目录、系统剪贴板或“记忆力”。保存成功、丢弃、应用退出或应用下次启动时必须清理相关缓存。

## 用户故事

1. 作为用户，我想框选后直接在原位置标注，以便操作不中断、不跳入另一个编辑窗口。
2. 作为用户，我想松开鼠标时立即固定选区，以便截图结果和我的框选动作保持一致。
3. 作为用户，我想选区之外恢复正常桌面显示，以便仍然看得到当前上下文。
4. 作为用户，我想编辑期间阻止选区外点击，以便不会误操作底层窗口。
5. 作为用户，我想框选时保持原有鼠标样式，以便界面变化尽量克制。
6. 作为用户，我想看到选区边框和实时尺寸，以便判断框选范围。
7. 作为用户，我想单击或极小框选不进入编辑，以便误操作后可以直接重选。
8. 作为多显示器用户，我想在任意显示器发起框选，以便不受主显示器限制。
9. 作为多显示器用户，我接受选区不能跨显示器，以便 V1 行为稳定可预期。
10. 作为用户，我想工具栏跟随选区且始终可见，以便不需要寻找操作入口。
11. 作为用户，我想直接保存未标注截图，以便标注层不会阻碍普通截图。
12. 作为用户，我想矩形圈出问题区域，以便 AI 能快速识别关注点。
13. 作为用户，我想用箭头表达方向和关联，以便 AI 理解界面元素之间的关系。
14. 作为用户，我想在截图上补充简短多行文字，以便说明画面无法表达的问题。
15. 作为用户，我想标注样式固定且醒目，以便无需先配置样式。
16. 作为用户，我想标注工具保持激活，以便可以连续添加同类标注。
17. 作为用户，我想撤销正在进行的错误操作，以便不会先生成无效标注。
18. 作为用户，我想连续撤销已完成标注，以便可以回到原始截图。
19. 作为用户，我不需要重做和对象编辑，以便第一版保持轻量。
20. 作为用户，我想只有点击保存才生成文件，以便未确认截图不会污染保存目录。
21. 作为用户，我想保存时看到明确的处理中状态，以便不会重复点击。
22. 作为用户，我想保存成功后收到无声音通知，以便确认结果而不被声音打扰。
23. 作为用户，我想保存失败后保留全部内容，以便修复目录后继续保存。
24. 作为用户，我想点击 `×` 直接丢弃，以便放弃截图不需要确认步骤。
25. 作为用户，我不想遮罩、边框、工具栏或鼠标出现在最终图片中。
26. 作为用户，我想截图时 Deskmon 保持原样，以便当前真实情况能被完整记录。
27. 作为用户，我不想截图写入剪贴板或“记忆力”，以便现有内容不被覆盖或扩散。
28. 作为用户，我想重复截图请求被忽略，以便不会叠出多个框选和编辑层。
29. 作为开发者，我想截图任务有一个完整生命周期，以便任务锁和临时数据能够可靠释放。
30. 作为开发者，我想标注使用截图原图坐标，以便 Retina 和不同缩放比例下导出位置一致。
31. 作为开发者，我想只排除截图工具自身窗口，以便宠物和其他 Deskmon 窗口仍能进入截图。

## 实现决策

- 用户可见入口仍叫“区域截图”，不新增“截图并标注”菜单项，也不提供跳过编辑的第二入口。
- 本 PRD 不再使用 `/usr/sbin/screencapture -i -s` 完成框选；原生框选器无法提供稳定的选区屏幕坐标，也无法承接原位置编辑。
- 使用按显示器建立的透明无边框覆盖层实现框选与编辑；这些覆盖层是实现细节，用户不能感知到新的普通窗口。
- 框选覆盖层负责遮罩、拖拽、尺寸标签和选区边框；编辑覆盖层负责冻结图像、标注、工具栏和错误反馈。
- 框选开始时覆盖所有显示器，首次有效按下决定起始显示器；选区坐标必须限制在该显示器内。
- 捕获时刻固定为有效选区松开鼠标的瞬间，不包含鼠标指针。
- 截图捕获层必须排除本次截图任务创建的全部覆盖层，但不得按应用整体排除 Deskmon 窗口。
- 框选阶段不主动改变鼠标指针样式；不播放框选完成、编辑或保存声音。
- 小于 `10 × 10 px` 的选区不创建编辑会话、不释放任务锁，继续等待用户重新框选或按 `Escape` 取消。
- 有效选区进入编辑后固定在原屏幕位置，不支持移动、缩放和二次裁剪。
- 冻结选区保持正常亮度，选区之外延续轻度暗色遮罩并保持实时变化；覆盖层继续接收并拦截点击，直到截图任务结束。
- 工具栏优先下方居中，其次上方居中；水平位置限制在显示器内，仍无空间时覆盖选区底部内侧。
- 工具栏初始无激活工具；矩形、箭头或文字被选择后保持激活，支持连续创建。
- 标注数据统一使用截图原图坐标；显示坐标、屏幕逻辑坐标和物理像素坐标必须明确转换。
- 矩形为透明填充的轮廓；箭头头部位于拖拽终点；文字支持多行且最多 200 字。
- 切换工具时，非空文字完成、空文字取消；其他未完成临时操作取消且不进入历史。
- 撤销优先取消当前临时操作，否则删除最近完成标注；没有历史和临时操作时按钮禁用。
- 完成后的标注不可单独编辑；V1 不提供重做。
- 标注主体统一使用红色和细白色外描边，不暴露样式设置。
- 标注绘制和文字输入必须裁切在截图边界内。
- 最终 PNG 使用原始截图物理像素尺寸合成；工具栏、边框、尺寸标签、错误反馈和其他 UI 永不参与合成。
- 一次截图只生成一个最终 PNG；没有标注时直接导出原始截图内容。
- 最终保存目录继续使用现有默认桌面或自定义目录偏好；目录不存在时自动创建。
- 文件名继续使用 `Deskmon_YYYY-MM-DD_HH-mm-ss.png`，时间取有效框选完成时刻，同名从 `_2` 开始避让。
- 最终路径只能在用户点击保存时创建；编辑期间不能提前在用户保存目录占位。
- 保存进行中禁用保存按钮；保存成功后关闭覆盖层、清理缓存、释放任务锁并发送无声音通知。
- 保存失败时保持编辑会话和任务锁，错误显示在工具栏下方。
- 目录修复只在用户点击对应动作后执行；修复完成不自动重试保存。
- 点击 `×` 直接清理并退出，不确认、不落盘、不通知。
- `Escape` 只保留为框选阶段的取消能力，不扩展为编辑快捷键。
- 截图任务锁从创建框选层开始，到框选取消、保存成功或丢弃时释放；重复请求静默忽略。
- 编辑会话只存在于运行期，不写入用户设置，应用重启不恢复未保存内容。
- 截图权限由 macOS CoreGraphics 系统预检结果判定，不能只根据 `screencapture` 的错误文字推断；权限已通过时，区域、坐标或临时文件错误必须显示真实原因。
- 开发包与正式包都必须使用稳定签名身份和固定 `com.sansan.deskmon` bundle id，避免重复构建或升级后系统仍显示旧授权、当前进程却无法截图。
- 从开发签名首次切换到正式发行签名时，可以要求当前测试用户重新授权一次录屏；之后同一发行身份的版本升级不应反复丢失权限。
- 默认桌面、自定义目录和目录修复沿用已有区域截图能力。

建议模块边界：

- 截图会话协调器：管理框选、编辑、保存、丢弃、任务锁和覆盖层生命周期。
- 屏幕覆盖层管理器：按显示器创建、定位和销毁透明覆盖层，处理跨屏边界与输入拦截。
- 屏幕捕获适配器：在指定显示器和物理像素区域捕获画面，只排除截图工具自身窗口。
- 标注状态模块：表达矩形、箭头、文字、临时操作和撤销历史。
- 坐标转换模块：处理屏幕逻辑坐标、显示器缩放、截图原图坐标和 Retina 像素映射。
- PNG 合成模块：把原始截图与标注确定性合成为最终 PNG，不感知窗口状态。
- 临时数据模块：管理内部缓存与异常清理，不接触最终保存目录。

## 测试决策

- 测试关注用户可见行为和稳定模块契约，不依赖具体 DOM 层级或覆盖层私有实现。
- 框选逻辑测试覆盖四个拖动方向、显示器边缘限制、负坐标显示器和 `10 × 10 px` 有效边界。
- 极小选区测试验证误操作后仍可继续框选，不生成编辑会话和最终文件。
- 多显示器测试验证鼠标按下所在显示器决定边界，拖动不能跨屏，工具栏留在同一显示器。
- 工具栏定位测试覆盖下方、上方、左右贴边和近全屏选区的内侧兜底位置。
- 捕获测试验证截图时刻为鼠标松开瞬间，并排除指针、遮罩、边框、尺寸标签和工具栏。
- 捕获测试验证只排除截图覆盖层，框选到宠物、设置窗口或“记忆力”时仍能捕获它们。
- 坐标测试覆盖 Retina、非 Retina、不同缩放比例、负坐标屏幕和多显示器混合缩放。
- 标注状态测试覆盖矩形、箭头、文字完成、连续同工具创建和切换工具。
- 文字测试覆盖多行、200 字限制、空白取消、点击其他位置完成和切换工具完成。
- 撤销测试覆盖取消临时操作、单步回退、连续回退、回到原图和空状态禁用。
- 边界测试验证全部标注被裁切在截图内，不覆盖桌面或工具栏。
- 视觉状态测试验证编辑中仅选区外变暗，非选区显示器保持整屏遮罩。
- PNG 合成测试验证输出保持原始物理像素尺寸，矩形为空心、箭头方向正确、文字位置稳定，并存在红色主体与白色描边。
- 无标注导出测试验证结果与冻结的原始截图视觉内容一致。
- 保存测试覆盖按钮防重复、成功生成单文件、无声音通知、失败保留状态和再次保存成功。
- 目录测试复用默认桌面、目录自动创建、英文命名和重名避让，并覆盖重新选择目录与恢复桌面后不自动保存。
- 丢弃测试验证无确认、无文件、无通知、缓存清理和任务锁释放。
- 会话测试覆盖框选取消、编辑期间重复请求静默忽略、应用退出清理和下次启动清理残留缓存。
- 权限 smoke test 覆盖首次拒绝、授权后重试、已有授权启动和同一正式签名身份覆盖升级，确保授权提示只在系统预检未通过时出现。
- 手工 smoke test 验证框选遮罩轻且不影响识别，选区边框与尺寸清晰，框选时鼠标样式不被切换。
- 手工 smoke test 验证松开后选区原地冻结并保持正常亮度，选区外持续变暗、实时变化但无法被误点击。
- 手工 smoke test 验证工具栏在各种边缘位置可见，且保存图片中不包含任何编辑 UI。
- 手工 smoke test 验证编辑期间宠物动画、移动、专注和其他 Deskmon 窗口持续正常。
- 手工 smoke test 验证框选、保存和通知全过程没有声音。

## 不在范围内

- 视频录制、GIF、动态 PNG、动态 WebP 或关键帧提取。
- 普通、可移动、可缩放的独立编辑窗口。
- 框选完成后直接保存或提供跳过编辑的第二入口。
- 原生 `screencapture -i -s` 框选交互、十字光标或窗口截图模式。
- 跨显示器选区、选区移动、选区缩放和二次裁剪。
- 全屏截图、整个显示器截图、窗口截图或空格切换窗口模式。
- 右下角浮动缩略图、延时预览或点击缩略图进入编辑。
- 颜色选择、色板、线宽、透明度、虚线和阴影设置。
- 字体、字号、粗体、斜体、对齐和富文本格式。
- 画笔、自由曲线、直线、椭圆、多边形和高亮笔。
- 裁剪、旋转、滤镜、色彩调整和图片压缩设置。
- 马赛克、模糊、敏感信息自动识别和自动打码。
- 编号工具、贴纸、图标、表情和步骤模板。
- 已完成标注的选择、移动、缩放、旋转、复制、层级调整和单独删除。
- 重做、历史面板、跨会话恢复和编辑草稿自动保存。
- 编辑快捷键、全局快捷键和自定义快捷键设置。
- 保存未标注原图副本或同时输出原图与标注图。
- JPG、PDF、SVG 或其他输出格式。
- 自动打开 Finder、预览图片或保存后继续编辑。
- 写入系统剪贴板、复制图片或复制标注文字。
- 截图历史、图库、搜索、收藏、删除管理和云同步。
- OCR、文字识别、自动生成说明、AI 自动标注或直接上传 AI。
- Windows 和 Linux 的截图标注实现。

## 进一步说明

- 本 V1 的目标是让“框选、标注、保存”成为一次连续的桌面操作，不是构建通用图片编辑器。
- “原地编辑”允许内部使用透明无边框窗口，但用户不能看到普通窗口标题栏、窗口切换或居中编辑页。
- “点击保存才文件化”指用户保存目录中只在保存时出现最终 PNG；受控内部缓存必须严格清理。
- 固定标注样式是范围控制手段；复杂背景可见性通过红色主体加白色描边解决，而不是增加样式面板。
- 原区域截图 PRD 仍负责权限、保存目录和命名等基础规则；与原生框选、立即保存和截图声音有关的旧规则由本 PRD 覆盖。
