## 阅读顺序

了解当前产品时，先读根目录 README，再读 011、008、002 和 003。桌宠基础行为按需补读 001、005 和 010。

追溯方案演变时，再读 004、006、007 和 009。旧文档中的实现状态、菜单名称和视觉参数可能只代表当时版本。

## 维护规则

- 新 PRD 使用递增编号，不覆盖旧文件。
- 新方案改变旧规则时，在新旧 PRD 顶部写清覆盖关系。
- 实现、合并或废弃后及时更新状态，不让“待实现”长期残留。
- README 只描述当前能力，不承载完整需求推导。
- 工程执行结果写入 `.ai/logs/`，长期约束写入 `.ai/keypoint.md`。
