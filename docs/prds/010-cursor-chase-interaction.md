# PRD：桌宠鼠标追逐互动

状态：已实现，待用户手感验收
目标 triage 标签：ready-for-agent
创建日期：2026-07-23
最后更新：2026-07-24
发布状态：已发布到 GitHub issue #19：https://github.com/l3ve/deskmon/issues/19

## 问题陈述

Deskmon 已经可以自主移动、休息、拖拽、悬停暂停、显示专注状态和完成提醒，但日常桌面状态下，宠物与鼠标之间仍缺少即时互动。用户移动鼠标经过宠物附近时，宠物不会注意、靠近或回应，因此角色更像按固定节奏移动的桌面图标，而不是会感知用户的小伙伴。

用户希望增加一种轻量、安静、可重复的“逗宠物”体验：当鼠标在宠物附近快速移动时，宠物会注意到鼠标并短暂追逐。互动必须足够生动，但不能让宠物持续跟随鼠标、跨越整个桌面、妨碍点击或拖拽，也不能破坏专注计时、截图、提醒展示和活动区域等现有行为。

这个需求最重要的约束是可操作性。用户把鼠标移向宠物时，通常可能准备拖拽或打开右键菜单；如果宠物在鼠标靠近时立即移动，就会形成“越想点，它越跑”的挫败感。因此追逐必须区分“在附近逗它”和“直接靠近操作它”，让宠物有生命感的同时仍然是一个稳定、可点击的桌面控件。

## 解决方案

### 互动主流程

Deskmon 在宠物可见、未暂停移动且处于普通桌面状态时，低频采样鼠标位置。鼠标在远处移动不会触发任何反应；只有进入宠物周围的感应外圈并达到快速移动阈值，才开始一次候选互动。

一次完整互动包含四个阶段：

1. **注意**：鼠标快速进入感应外圈后，宠物进入约 100ms 的意图判断。宠物原地轻抬约 5 个逻辑像素，表示已经注意到鼠标。
2. **追逐**：判断结束时，如果鼠标仍在感应外圈快速移动，宠物使用现有 `run` 动画进行短时间实时追踪。
3. **观察**：宠物进入点击安全内圈、达到追逐上限、鼠标跑出可追范围或宠物到达活动区域边缘后，停止移动并朝最后有效鼠标方向保持 `idle` 约 1 秒。
4. **冷却**：观察结束后进入 2 秒冷却。冷却期间宠物可以恢复日常活动，但不能再次触发鼠标追逐。

追逐不是持续跟随模式。单次追逐最长约 1.2 秒，最远移动约 180–220 个归一化逻辑像素；达到任一上限就结束。追逐目标约每 80–120ms 根据最新鼠标位置更新，因此宠物能对短时间内的方向变化作出反应，但不会逐帧紧贴鼠标或产生高频窗口调用。

### 感应范围与点击安全

感应范围相对当前宠物窗口尺寸计算，并换算到统一的逻辑距离，避免小、中、大尺寸以及 Retina、多显示器缩放下的体感明显不同。

- 感应外圈的实机调校基线约为当前宠物窗口宽度的 3.5 倍；中等尺寸下约等于 350 个逻辑像素。
- 点击安全内圈的首轮调校基线约为当前宠物窗口宽度的 1.2 倍；中等尺寸下约等于 120 个逻辑像素。
- 快速移动的实机调校基线是：鼠标在滚动 400ms 窗口内累计移动约 90 个归一化逻辑像素。

鼠标从外部快速进入感应外圈会触发“注意”，但不会立即移动宠物。约 100ms 的判断窗口用于区分两种意图：

- 鼠标继续在外圈移动、横向掠过或从宠物附近快速离开：进入追逐。
- 鼠标直接进入点击安全内圈：取消候选追逐，宠物原地等待操作。

鼠标从宠物旁快速向外移动时，离开意图已经明确，可以跳过大部分判断时间并快速开始追逐。鼠标朝宠物靠近时始终优先保护点击，不允许宠物因接近动作突然逃离。

DOM `pointerenter`、`pointerdown` 和已经存在的拖拽流程拥有更高优先级：

- 鼠标实际进入宠物窗口时，立即取消注意或追逐并暂停自主移动。
- 左键按下后立即交给现有 pointer capture 和拖拽判定处理。
- 右键菜单继续按现有方式打开，追逐不能抢占或延迟菜单事件。
- 鼠标离开宠物窗口后的短暂保护期内不能立刻重启追逐，避免菜单或拖拽结束后宠物马上移动。

### 追逐表现

追逐复用现有 `run` sprite，不新增宠物 mood、sprite sheet、粒子、气泡或素材。追逐开始和实时更新目标时，宠物根据水平移动方向使用现有左右朝向帧。

“注意”阶段只做轻量但可辨认的垂直位移：宠物原地抬起约 5 个逻辑像素并落回，不缩放窗口、不改变窗口位置、不写入设置。该动作使用现有画布绘制结果，不修改 PNG 素材，也不使用 `celebrate`；即使用户随后直接进入安全圈并取消追逐，仍然可以看到宠物注意到了鼠标。

追逐速度基于现有活动档位轻微调整：

- 安静：追逐速度略低于标准档，但触发规则、时长和冷却一致。
- 标准：作为默认手感调校基线。
- 活泼：追逐速度略高于标准档，但不能延长追逐时间或缩短冷却。

三个档位都能触发互动，不能因为安静档而完全没有反馈，也不能因为活泼档而更频繁地误触发。普通自主移动目标在追逐期间暂停；观察和冷却结束后，再恢复现有自主移动节奏。

### 活动区域与状态优先级

宠物追逐始终服从现有活动区域和可见工作区限制。鼠标位于活动区域之外时，宠物最多追到允许位置的边缘，然后进入观察，不越界、不穿过 Dock 或菜单栏，也不为追逐临时扩大活动区域。

以下状态禁用或中断鼠标追逐：

- 宠物已隐藏。
- 用户已选择暂停移动。
- 专注计时或休息计时正在进行。
- 专注完成、休息中、休息完成等中央交接正在展示。
- 巨型庆祝或外部宠物提醒正在进入、停留或恢复。
- 用户正在拖拽宠物。
- 区域截图或原地标注任务正在进行。
- 宠物窗口正在执行其他临时展示。

这些状态结束后，不能补播期间错过的鼠标互动；只有新的有效鼠标移动才能重新触发。追逐不会暂停、取消或修改任何业务状态，也不能覆盖专注、提醒、截图和直接用户操作的视觉优先级。

### 安静与设置边界

鼠标追逐完全静音，不播放宠物音效、不调用系统提示音，也不发送系统通知。

第一版不增加独立开关、灵敏度设置、追逐距离设置或新的菜单项。用户已有的“暂停移动”同时关闭自主移动和鼠标追逐；现有安静、标准、活泼档位只轻微影响追逐速度。

本轮不增加左键单击反馈。左键继续只用于现有单击/拖拽判定，右键继续打开原生宠物菜单。摸头、弹跳、亲密度或其他点击互动需要另行设计，不能混入本 PRD。

## 实现决策

- 鼠标追逐作为普通桌面状态下的新交互能力，不新增持久化设置、菜单入口、权限或 schema。
- 复用现有全局鼠标位置查询能力，不注册系统级鼠标监听器，不引入新的 macOS 权限。
- 鼠标采样只在宠物可见、未暂停、没有专注计时、没有拖拽、没有截图任务和没有临时展示时运行。
- 鼠标互动状态封装为独立深模块。模块接收时间、鼠标位置、宠物位置与尺寸、活动区域、活动档位和阻塞状态，输出注意、追逐、观察、冷却或空闲结果。
- 独立状态模块不直接调用 Tauri、不访问 DOM、不移动窗口，也不感知具体 sprite；控制器负责把状态结果映射到窗口移动和现有 mood。
- 状态模块使用空闲、候选注意、追逐、观察和冷却五个内部阶段；这些是交互协调状态，不增加用户可见的宠物 mood。
- 感应外圈实机调校基线为当前宠物窗口宽度约 3.5 倍，点击安全内圈保持约 1.2 倍。
- 距离、速度和累计移动阈值使用归一化逻辑坐标计算；窗口移动仍使用现有 native 物理坐标，并沿用当前坐标换算边界。
- 快速移动实机调校基线为滚动 400ms 内累计约 90 个归一化逻辑像素；实现保留集中常量以供实机调校，不暴露用户设置。
- 从外部进入感应外圈后先进入约 100ms 候选注意；候选期间提供清晰但克制的视觉反馈，不改变宠物窗口位置。
- 鼠标在候选期间进入点击安全内圈，或触发 DOM `pointerenter`、`pointerdown`，立即取消候选和追逐。
- 鼠标从宠物附近快速向外移动时允许缩短候选等待，但仍必须满足快速移动和感应范围规则。
- 追逐期间约每 100ms更新一次目标，窗口移动继续沿用现有节流与 in-flight 合并机制，不能为鼠标每个原始采样点单独发起窗口调用。
- 单次追逐最长约 1.2 秒，最大归一化移动距离首轮基线为 200px；时长、距离、进入内圈、失去有效目标或到达活动区域边缘任一成立即结束。
- 追逐目标限制在现有活动区域和可见工作区内；鼠标位于区域外时使用边缘内的合法目标，不改变活动区域设置。
- 追逐复用现有 `run`，朝向继续使用现有左右方向帧；注意和观察复用现有 `idle`。
- 注意动作通过绘制阶段的短暂整数像素垂直偏移完成，不缩放或移动原生窗口，不修改 sprite PNG。
- 安静、标准、活泼只对追逐速度应用轻微倍率；触发阈值、追逐上限、观察时间和冷却保持一致。
- 观察阶段约 1 秒，保持最后有效水平朝向；观察结束后进入 2 秒冷却。
- 冷却只阻止新的鼠标追逐，不阻止点击、拖拽、右键菜单和普通自主移动。
- 追逐在移动优先级中低于拖拽、截图、专注流程、巨型庆祝、外部提醒和所有临时展示，高于普通自主移动。
- 专注计时和休息计时运行期间完全禁用追逐，不能用 `run` 覆盖 `timer-waiting`。
- “暂停移动”同时停止普通自主移动和鼠标追逐；被暂停或其他高优先级状态中断的互动直接清空，不排队恢复。
- 追逐产生的普通宠物位置继续沿用现有位置更新与低频持久化策略，不引入临时展示位置语义，也不增加额外磁盘写入频率。
- 本功能完全静音，不发送系统通知，不影响记忆力、变量库、截图结果或专注状态。
- 第一版不增加独立开关、灵敏度设置、菜单项或左键单击反馈。

建议模块边界：

- 鼠标互动状态模块：管理采样历史、快速移动判定、候选注意、追逐、观察、冷却和中断。
- 宠物控制器协调层：查询鼠标位置、提供当前阻塞状态、消费互动结果并协调普通移动。
- 互动几何策略：计算归一化距离、内外圈、合法追逐目标、最大位移和活动区域边缘。
- 轻量视觉反馈：在绘制阶段应用注意动作的整数像素偏移，不感知窗口或业务状态。

## 测试决策

- 测试关注用户可见行为和稳定模块契约，不断言私有字段、精确 requestAnimationFrame 次数或具体 sprite 帧号。
- 鼠标互动状态模块使用现有 Vitest 测试方式进行纯逻辑测试，不依赖真实 Tauri 窗口或 macOS 鼠标。
- 快速移动测试覆盖低于阈值不触发、400ms 内累计距离达到阈值触发、过期采样不参与计算。
- 感应范围测试覆盖外圈之外不触发、进入外圈进入候选、不同宠物尺寸保持相对范围一致。
- 点击安全测试覆盖候选期间进入内圈取消、追逐期间进入内圈停止、pointer enter/down 中断后不继续移动。
- 意图判断测试覆盖从外部快速移入后直接点击不追逐、停留在外圈继续移动会追逐、快速向外移动缩短等待。
- 追逐测试覆盖目标按最新鼠标位置更新、左右朝向随水平移动变化、普通自主移动目标在追逐期间不生效。
- 上限测试覆盖 1.2 秒时长、最大移动距离、鼠标失去有效目标和到达活动区域边缘都会结束追逐。
- 活动区域测试覆盖普通边缘、负坐标显示器和鼠标位于区域外时目标仍被限制在合法范围。
- 观察测试验证追逐结束后保持约 1 秒、保留最后方向并映射到 `idle`。
- 冷却测试验证观察后 2 秒内不能再次触发，冷却结束后的新快速移动可以触发。
- 活动档位测试验证安静低于标准、活泼高于标准的追逐速度关系，同时三档共享触发、时长和冷却规则。
- 优先级测试覆盖暂停移动、专注计时、休息计时、拖拽、截图、巨型庆祝、外部提醒和临时展示会禁用或中断互动。
- 中断测试验证高优先级状态结束后不会补播旧追逐，必须等待新的有效鼠标动作。
- 采样协调测试验证阻塞状态下不发起鼠标查询，普通状态下查询频率不超过约定上限。
- 控制器集成测试或可测试协调函数验证互动结果正确映射到 `idle`、`run`、朝向和窗口目标，不锁定 DOM 结构。
- 现有拖拽和悬停行为应继续通过构建与桌面 smoke test 验证，避免追逐改动破坏 pointer capture。
- 手工 smoke test 应验证从外部快速移入、外圈横向移动、从宠物旁快速移开和直接进入点击四条路径。
- 手工 smoke test 应验证注意动作可见但克制，短追逐不会滑行、抖动、频繁折返或跨桌面。
- 手工 smoke test 应验证左键拖拽、右键菜单、悬停暂停和菜单关闭后的短保护期。
- 手工 smoke test 应观察安静、标准、活泼以及小、中、大尺寸下的相对手感。
- 手工 smoke test 应覆盖 Retina、多显示器缩放、活动区域边缘和负坐标显示器。
- 手工 smoke test 应验证专注、截图和外部提醒期间不会因鼠标移动触发追逐。
- 手工 smoke test 应确认整个互动完全静音且不产生系统通知。

## 不在范围内

- 持续跟随鼠标或把鼠标变成宠物的长期移动目标。
- 鼠标移动到桌面任意位置都触发追逐。
- 追逐跨越整个屏幕、跨越活动区域或突破可见工作区。
- 宠物逃离鼠标、躲闪、碰撞、反弹、寻路或复杂物理轨迹。
- 新的宠物 mood、sprite sheet、皮肤、粒子、彩纸、气泡或视觉素材。
- 使用 `celebrate` 作为普通鼠标互动反馈。
- 音效、系统提示音、语音、系统通知或震动反馈。
- 独立追逐开关、灵敏度、感应范围、速度、时长或冷却设置。
- 新的菜单项、托盘入口、快捷键或全局手势。
- 左键单击反馈、摸头、喂食、亲密度、心情、养成或成长系统。
- 修改现有拖拽阈值、右键菜单行为或悬停暂停语义。
- 修改现有活动区域设置、宠物尺寸设置或活动档位结构。
- 专注计时期间的追逐或新的计时专属互动。
- 截图和标注期间的鼠标追逐。
- 外部提醒、巨型庆祝和中央交接期间的追逐。
- 记录鼠标轨迹、保存互动历史、统计互动次数或同步数据。
- 系统级事件监听、新的 macOS 辅助功能权限或输入监控权限。
- AI 对话、根据用户行为推断心情或自适应学习鼠标习惯。
- Windows 和 Linux 的独立交互规则。

## 进一步说明

- 350px 外圈、120px 内圈、90px/400ms 快速移动阈值、100ms 判断、1.2 秒追逐、200px 最大位移、1 秒观察和 2 秒冷却是当前实机调校基线；允许在不改变产品语义的前提下小幅调整。
- 可点击性优先于追逐响应速度。任何调参只要让用户更难拖拽或打开右键菜单，都应回退。
- 第一版的成功标准不是“宠物尽可能多地响应”，而是用户主动在附近快速移动鼠标时能稳定逗到它，普通使用时又很少误触发。
- 视觉验收必须在真实 macOS 桌面完成；单元测试和构建只能证明状态与边界没有损坏，不能证明互动是否自然。
