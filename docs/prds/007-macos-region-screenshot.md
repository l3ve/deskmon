目标 triage 标签：ready-for-agent
创建日期：2026-07-14
最后更新：2026-07-14
发布状态：已发布到 GitHub issue #13：https://github.com/l3ve/deskmon/issues/13

实现结果：已接入菜单栏和宠物右键菜单、原生矩形框选、PNG 文件直存、目录设置、任务互斥、权限引导与结果通知；核心路径与结果分类已覆盖单元测试。

## 问题陈述

Deskmon 已经通过菜单栏和宠物右键菜单提供桌宠控制、专注计时和“记忆力”等轻量能力，但用户在桌面工作时仍需要离开 Deskmon，使用系统快捷键或其他工具完成区域截图。对于只想快速框选一块内容并保存为文件的用户，这段操作缺少一个与现有 Deskmon 入口一致的触发方式。

截图能力容易膨胀为完整截图工具，包括全屏截图、窗口截图、标注、预览、历史、云同步和快捷键管理。本次需求并不是替代 macOS 截图工具，而是让 Deskmon 提供一个轻量入口：用户主动触发后，直接进入系统原生矩形框选，完成后把 PNG 文件保存到约定目录。

截图过程还必须尊重 Deskmon 当前状态。用户不希望 Deskmon 为了截图而隐藏宠物、隐藏窗口、暂停动画、暂停专注流程或改变任何界面状态。Deskmon 可以出现在用户主动选择的截图区域中；截图能力只负责发起和保存，不接管桌面场景。

## 解决方案

Deskmon 在菜单栏菜单和宠物右键菜单中各增加一个一级入口“区域截图”。用户点击后，应用调用 macOS 原生交互式截图能力，并将交互锁定为矩形区域框选：

1. 用户通过十字光标拖出矩形区域。
2. 松开鼠标后完成截图。
3. `Escape` 取消当前截图。
4. 不允许切换为整窗截图。
5. 截图不包含鼠标指针。
6. 截图成功时保留 macOS 系统截图声音。

截图期间 Deskmon 保持完全正常：宠物继续活动，设置窗口和“记忆力”窗口保持原样，专注或休息计时继续推进，中央交接对话框也不被隐藏或暂停。如果用户把 Deskmon 框入选区，Deskmon 会正常出现在图片中。

截图结果直接保存为 PNG 文件，不写入系统剪贴板，也不在 Deskmon 内生成预览、历史或图库。默认保存目录是当前用户的桌面。用户可以在偏好设置中新建的“区域截图”区域选择其他目录，或恢复为默认桌面。该设置区位于“专注计时”和“活动区域”之间。

文件名使用英文、数字和 ASCII 分隔符，格式为：

`Deskmon_YYYY-MM-DD_HH-mm-ss.png`

例如：`Deskmon_2026-07-14_15-32-08.png`。如果同一秒内已经存在同名文件，依次追加 `_2`、`_3` 等序号，绝不覆盖已有文件。

截图开始前，Deskmon 检查目标目录。目录不存在时自动创建；创建失败或目录不可写时，不进入框选，并显示原生错误提示，让用户重新选择目录或恢复为桌面。用户完成框选后如果因磁盘空间、权限变化等原因保存失败，Deskmon 发送“截图保存失败”的系统通知，不创建空文件、不自动修改目录，也不自动重新开始截图。

截图成功后，Deskmon 发送系统通知，明确图片已保存并显示文件名。成功后不自动打开图片、不打开 Finder、不显示预览。用户取消截图时保持安静，不显示成功或失败通知，也不播放截图声音。

首次截图由 macOS 正常处理屏幕与系统录音权限请求。如果权限被拒绝或截图失败可明确归因于权限，Deskmon 显示原生提示框，提供“打开系统设置”和“取消”，但不未经确认直接跳转系统设置。

同一时间只允许一个截图任务。截图进行中不改变菜单项外观；如果极端情况下再次收到截图请求，Deskmon 静默忽略重复请求，不叠加第二个系统框选层，也不打断当前截图。

多显示器行为完全沿用 macOS 原生框选能力。Deskmon 不限制从哪块屏幕开始，不自行修改选区，也不额外约束跨屏选择。

## 实现决策

- 本能力第一版仅支持 macOS，不为 Windows 或 Linux 预先实现另一套框选交互。
- 用户可见命名统一为“区域截图”。
- 菜单栏菜单和宠物右键菜单各增加一个一级“区域截图”入口。
- 不修改宠物左键行为，不增加全局快捷键。
- 使用 macOS 原生交互式截图能力，并强制使用鼠标矩形选择模式；不允许切换为窗口选择。
- 截图输出固定为 PNG，不提供格式或压缩质量设置。
- 截图不包含鼠标指针，成功时不关闭系统截图声音。
- `Escape` 被视为用户取消，而不是失败；取消不产生文件、通知或额外声音。
- 截图流程不隐藏、移动、暂停或调整任何 Deskmon 窗口和宠物状态。
- 截图流程不暂停专注、休息、动画、菜单状态或中央交接流程。
- Deskmon 可以出现在截图结果中，是否包含由用户选区决定。
- 多显示器选择完全沿用系统原生行为，不自行限制屏幕或修正选区。
- 截图结果直接写入文件系统，不写入系统剪贴板。
- 截图不进入“记忆中”、笔记本或变量库，现有“记忆力”仍保持纯文本边界。
- 默认保存目录为系统解析出的当前用户桌面目录，不能硬编码用户名或本机绝对路径。
- 用户可以保存一个自定义截图目录；未设置自定义目录代表使用默认桌面。
- 设置页新增独立“区域截图”区域，位于“专注计时”和“活动区域”之间。
- 设置区显示当前有效目录，并提供“选择文件夹”和“恢复默认”两个操作。
- 目录选择使用现有原生对话框能力，限定为选择文件夹；不提供手动路径输入。
- “恢复默认”清除自定义目录，而不是把当前桌面绝对路径固化到设置中。
- 旧设置缺少截图字段时必须自动使用默认桌面，升级过程不需要迁移操作。
- 开始框选前解析有效保存目录；目录不存在时递归创建。
- 目录创建失败或不可写时不启动原生框选，并显示原生错误提示，提供重新选择或恢复桌面。
- 文件名固定为 `Deskmon_YYYY-MM-DD_HH-mm-ss.png`，使用本地时间。
- 同名冲突从 `_2` 开始递增选择可用文件名，任何情况下都不覆盖已有文件。
- 截图协调器是唯一流程入口，负责运行期单任务锁、目录准备、文件名分配、原生截图调用和结果反馈。
- 截图执行器只封装 macOS 原生截图调用，并返回成功、取消或失败，不直接修改设置或发送通知。
- 文件路径与命名逻辑作为独立深模块，封装默认目录解析、目录创建、可写性检查和冲突规避。
- 同一时间只允许一个截图任务；重复请求在内部静默忽略，不改变菜单项文本或可用状态。
- 成功以原生截图调用完成且目标 PNG 文件有效存在为准。
- 成功后发送系统通知，通知包含“截图已保存”和实际文件名。
- 成功后不自动打开图片、Finder、预览窗口或任何 Deskmon 管理界面。
- 用户完成框选后的非取消错误发送“截图保存失败”系统通知，不自动重试、不改目录、不保留空文件。
- 权限由 macOS 在首次调用时正常请求；权限相关失败使用原生说明提示，提供“打开系统设置”和“取消”。
- 未经用户点击“打开系统设置”，Deskmon 不主动跳转系统设置。
- 截图任务锁仅存在于内存，任务完成、取消或失败后都必须释放。
- 原生截图调用不能阻塞 Tauri 主事件循环，避免宠物动画、计时和其他窗口失去响应。

建议拆分的深模块或边界：

- 截图协调器：以一个稳定入口接收截图请求，串联路径准备、任务锁、执行器和反馈策略，并返回可观察结果。
- 截图路径服务：负责默认桌面解析、自定义目录处理、缺失目录创建、可写性检查、时间命名和冲突序号。
- 原生截图执行器：隔离 macOS 命令参数、进程退出和取消识别，向上只暴露成功、取消、失败三类语义。
- 截图偏好设置：集中管理可选自定义目录和旧设置兼容，不把运行期忙碌状态写入持久设置。
- 截图反馈适配器：将成功、保存失败、目录异常和权限异常映射为系统通知或原生提示。

## 测试决策

- 测试关注用户可见行为和模块契约，不依赖私有字段、具体线程实现或原生进程内部细节。
- 截图路径服务是主要单元测试边界，使用临时目录验证目录创建、文件命名和冲突规避。
- 路径测试应验证默认目录来自系统桌面解析，而不是硬编码本机路径。
- 路径测试应验证自定义目录不存在时可以递归创建。
- 路径测试应验证创建失败或目录不可写时在启动截图前返回明确错误。
- 文件名测试应验证英文前缀、完整本地时间格式和 `.png` 扩展名。
- 文件名测试应验证首个文件无序号，冲突时依次使用 `_2`、`_3`，并且不覆盖现有文件。
- 截图偏好设置测试应覆盖新安装默认使用桌面。
- 截图偏好设置测试应覆盖旧设置缺少截图字段时仍可加载并使用桌面。
- 截图偏好设置测试应覆盖保存自定义目录和恢复默认。
- 截图协调器使用可替换执行器测试，不在单元测试中拉起真实系统框选。
- 协调器测试应覆盖成功后发送一次成功反馈并释放任务锁。
- 协调器测试应覆盖取消后无通知、无错误，并释放任务锁。
- 协调器测试应覆盖保存失败后发送失败反馈，不自动重试，并释放任务锁。
- 协调器测试应覆盖截图进行中的重复请求被静默忽略。
- 协调器测试应覆盖目录准备失败时不调用原生截图执行器。
- 执行器契约测试应验证原生结果能够稳定映射为成功、取消和失败三类语义。
- 设置模块现有的默认值、规范化和旧配置兼容测试可作为偏好测试先例。
- 专注流程状态机现有的纯逻辑测试可作为深模块和替身测试先例。
- 菜单栏和宠物右键菜单需要手工 smoke test，确认两个入口都能启动同一截图流程。
- 原生框选需要手工 smoke test，确认只能矩形选择、`Escape` 可取消、空格不能切换整窗、结果不包含鼠标且保留系统声音。
- 保存行为需要手工 smoke test，确认默认桌面、自定义目录、目录自动创建、英文文件名和重名序号。
- 反馈行为需要手工 smoke test，确认成功通知、取消无提示、保存失败通知和权限引导。
- 桌面状态需要手工 smoke test，确认截图期间宠物、设置、“记忆力”、专注计时和中央交接均不被改变。
- 多显示器需要手工 smoke test，确认在不同屏幕开始框选时沿用系统行为。

## 不在范围内

- 自动截图、定时截图或专注期间自动截图。
- 全屏截图。
- 整个显示器截图。
- 窗口截图和空格切换窗口模式。
- 屏幕录制、视频、音频录制或点击可视化。
- 包含鼠标指针。
- JPG、PDF、TIFF 或其他输出格式。
- 图片质量、缩放比例、分辨率或压缩设置。
- 写入系统剪贴板。
- 同时保存文件并复制到剪贴板。
- 截图预览、浮动缩略图和自动打开图片。
- 自动打开 Finder 或“打开截图文件夹”菜单。
- 截图标注、裁剪、打码、OCR、文字识别或颜色拾取。
- 截图历史、图库、搜索、收藏和删除管理。
- 将截图保存到“记忆中”、笔记本或变量库。
- 云同步、上传、分享链接和第三方图床。
- 截图文件加密。
- 自定义文件名前缀、时间格式或序号规则。
- 自定义截图声音或关闭系统截图声音的设置。
- 全局快捷键和快捷键设置。
- 截图倒计时和延迟截图。
- 截图时隐藏 Deskmon、暂停宠物、暂停计时或修改任何窗口状态。
- 截图时禁用或改写菜单项外观。
- 跨平台截图实现；Windows 和 Linux 不在第一版范围内。
- 自制全屏遮罩、框选层或 Deskmon 品牌化截图界面。

## 进一步说明

- 这项能力是 Deskmon 对 macOS 原生区域截图的轻量入口，不是完整截图应用。
- “不改变 Deskmon”是明确产品决策，不是实现缺失。用户可以主动决定是否把宠物或 Deskmon 窗口框入截图。
- 默认桌面应通过系统路径 API 动态解析，文档中的“桌面”是用户可见语义，不代表固定字符串路径。
- 自定义目录是普通用户偏好；截图内容和截图历史不写入 Deskmon 配置。
- 原生框选、系统权限弹窗和系统截图声音属于 macOS 交互，自动化测试只验证适配层契约，最终仍需要本机手工验收。
