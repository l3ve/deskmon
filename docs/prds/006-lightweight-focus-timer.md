# PRD：Deskmon 轻量专注计时器

状态：已实现，并已按实际落地交互回写
目标 triage 标签：ready-for-agent（发布时）
创建日期：2026-07-09
最后更新：2026-07-09
发布状态：已发布到 GitHub issue #9：https://github.com/l3ve/deskmon/issues/9

## 问题陈述

Deskmon 当前已经有一个简单倒计时能力：用户可以从菜单栏或宠物右键菜单启动固定时长倒计时，计时中查看剩余时间或取消，到点后收到 macOS 通知，并在宠物可见时看到巨型庆祝动效。

这个能力已经形成了“桌宠陪你等待”的基础闭环，但它仍然偏固定：时长只支持内置的 `1 / 5 / 10 / 25` 分钟，完成提示文案固定，无法区分“专注结束”和“休息结束”。用户现在希望把它优化成更贴近日常使用的轻量专注计时器：保留 Deskmon 的克制桌宠体验，同时支持可配置的专注时长、休息时长和完成提示。

这次优化不能把 Deskmon 扩成完整番茄时钟或任务管理工具。用户明确接受“轻量专注”的方向：有专注段，有休息段，有提示文案，有休息结束声音，但不做自动循环、历史统计、任务列表、延后提醒、新窗口或新的宠物形象。

## 解决方案

将用户可见的“计时器”体验升级为“专注计时”。菜单里默认提供三个专注快捷时长和一个休息时长：

- 专注 5 分钟
- 专注 25 分钟
- 专注 45 分钟
- 休息 5 分钟

三个专注时长和一个休息时长都可以在设置窗口里修改。专注时长必须是 3 个互不重复的整数分钟，保存后按分钟数从小到大展示；休息时长是单个整数分钟。所有自定义时长范围均为 1 到 180 分钟。

计时仍然保持单计时器规则。未计时时，菜单展示可启动项；计时中，菜单只展示当前状态、剩余时间和取消入口。专注中显示“专注中：还剩 MM:SS”，休息中显示“休息中：还剩 MM:SS”。

完成提示分成两条用户可配置文案：

- 专注结束提示，默认“专注结束，休息一下吧”
- 休息结束提示，默认“休息结束，回来继续吧”

提示文案只用于 macOS 系统通知正文，不显示在宠物旁边，也不弹出新的应用窗口。文案不允许保存为空；清空或只输入空白时恢复默认文案。每条最多 80 个字符，不支持换行，保存时将换行压成空格。

专注结束和休息结束都复用现有宠物等待形态和巨型庆祝动效，不新增宠物状态、不新增休息形象。差异只体现在通知语气和声音上：

- 专注结束：发送专注结束文案通知，不主动播放声音，宠物可见且未被拖拽时播放巨型庆祝。
- 休息结束：发送休息结束文案通知，并在“休息结束播放声音”开启时播放 macOS 系统声音，宠物可见且未被拖拽时播放巨型庆祝。

“休息结束播放声音”是设置窗口里的一个开关，默认开启。声音只作为休息结束提醒的一部分，不引入宠物动作音效、不新增音频文件、不提供铃声选择。通知展示仍尊重 macOS 通知权限和勿扰模式；声音播放使用 macOS 内置系统声音文件，尊重系统输出音量。

设置变化只影响下一次新计时。启动一次计时时，Deskmon 捕获本次计时的类型、时长、完成文案和声音策略；如果用户在计时过程中修改设置，当前计时不变。

设置页使用桌面应用式偏好布局：左侧保留 Deskmon 摘要，右侧按“桌宠行为 / 专注计时 / 活动区域”纵向排列并可滚动。设置窗口默认尺寸需要能容纳主要配置，不能让“专注计时”被默认首屏挤没；小窗口时允许摘要区域折叠到顶部。

专注计时的分钟输入需要兼顾手动输入和连续点击。控件不依赖浏览器原生 `number` spinner，而使用自定义紧凑步进器：数值输入框旁放上下箭头，连续点击不应选中文本或导致按钮失效，同时仍支持键盘输入、回车保存和 Escape 恢复。

专注结束后不自动开始休息，也不进入“建议中”残留状态。完成后直接回到未计时状态，用户想休息时手动启动“休息 5 分钟”。休息结束后也不自动开始下一轮专注。

## 实现决策

- 本 PRD 是对现有倒计时能力的轻量专注化，不是完整番茄时钟。
- 用户可见命名统一使用“专注计时”。
- 代码内部可以继续使用既有计时器概念，避免为了文案重命名做大范围无意义改动。
- 计时模型需要区分两种类型：专注和休息。
- 默认专注快捷时长为 5 / 25 / 45 分钟。
- 默认休息时长为 5 分钟。
- 专注快捷时长固定为 3 个，不新增更多菜单项。
- 专注快捷时长可以在设置中修改。
- 休息时长可以在设置中修改。
- 所有自定义时长均为整数分钟。
- 自定义时长范围为 1 到 180 分钟。
- 3 个专注快捷时长不允许重复。
- 3 个专注快捷时长保存后自动升序展示。
- 设置窗口新增一个“专注计时”区域，不重做整个设置页。
- “专注计时”区域包含 3 个专注时长、1 个休息时长、2 条完成提示文案和 1 个休息结束声音开关。
- 设置页最终采用左侧摘要栏加右侧纵向面板列表的桌面偏好布局。
- 设置窗口默认内尺寸为 920 x 720，用于容纳桌宠行为、专注计时和活动区域。
- 设置页摘要栏只在窄窗口下切到顶部堆叠，避免常规尺寸下过早挤压内容。
- 设置页右侧内容区必须可纵向滚动，不能用固定高度网格压缩“桌宠行为”“专注计时”或“活动区域”卡片。
- 活动区域作为普通列表项展示，预览 canvas 使用有限可视高度，不能挤掉上方设置。
- 分钟控件使用自定义紧凑步进器，不依赖原生 `input type="number"` spinner。
- 自定义步进器需要支持手动输入、上下箭头增减、回车提交和 Escape 恢复。
- 连续点击步进箭头时不能选中输入框文本，也不能因为焦点或保存重绘导致按钮失效。
- 步进器视觉上使用整体控件，避免分散的加号/减号小按钮。
- 专注结束提示默认文案为“专注结束，休息一下吧”。
- 休息结束提示默认文案为“休息结束，回来继续吧”。
- 提示文案只用于 macOS 通知正文。
- 通知标题继续使用 Deskmon。
- 提示文案保存时需要去掉首尾空白。
- 提示文案为空或全空白时恢复默认文案。
- 提示文案最多 80 个字符。
- 提示文案不支持换行，保存时将换行压成空格。
- “休息结束播放声音”默认开启。
- 只有休息结束会播放系统声音。
- 专注结束不主动播放声音。
- 休息结束声音使用 macOS 内置系统声音文件单独播放，不依赖通知插件的声音参数。
- 实现不能新增音频文件。
- 实现不能新增铃声选择器。
- 实现不能绕过 macOS 系统输出音量。
- macOS 通知权限和勿扰模式影响通知展示；休息结束声音只作为轻量补充提醒。
- 未计时时，菜单展示 3 个专注入口和 1 个休息入口。
- 计时中，菜单不展示新的开始入口。
- 专注计时中，菜单显示“专注中：还剩 MM:SS”和取消入口。
- 休息计时中，菜单显示“休息中：还剩 MM:SS”和取消入口。
- 取消计时不触发通知、声音或巨型庆祝。
- 同一时间仍然只允许一个计时。
- 启动计时时需要捕获本次计时的类型、时长、完成文案和声音策略。
- 设置变化只影响下一次计时，不影响当前正在跑的计时。
- 设置页连续保存需要基于本地 draft preferences 合并，避免后一个保存请求用旧快照覆盖前一个字段。
- 配置需要持久化，退出重开后保留。
- 正在跑的计时不跨重启恢复。
- 专注中和休息中都复用现有等待形态。
- 专注结束和休息结束都复用现有巨型庆祝动效。
- 不新增新的宠物 mood。
- 不新增休息专用角色形象。
- 宠物隐藏或拖拽中完成计时，继续沿用现有规则：只发送通知，不强行显示宠物，不补触发庆祝。
- 专注结束后直接回到未计时状态。
- 休息结束后直接回到未计时状态。
- 专注结束不自动开始休息。
- 休息结束不自动开始下一轮专注。
- 完成后不保留“建议休息中”或类似残留状态。
- 不新增专注计时窗口。
- 不新增任务列表、历史统计、今日番茄数或连续番茄。
- 不新增延后提醒或重复提醒。

建议拆分的深模块或边界：

- 计时偏好模型：集中封装默认值、持久化结构、兼容旧设置和校验规则。
- 计时启动模型：以简单接口接收计时类型和时长，生成本次计时的不可变完成策略。
- 计时菜单模型：根据未计时、专注中、休息中生成原生菜单展示文案，并保持剩余时间项可更新。
- 完成通知策略：根据计时类型决定通知正文、是否播放休息结束声音，以及如何处理通知权限限制。
- 设置页专注计时区域：只负责编辑偏好，不承载正在运行的计时状态；分钟控件需要封装手动输入和步进交互。
- 设置页布局：摘要栏、右侧面板列表和活动区域预览需要共同满足默认窗口可用、小窗口可滚动。
- 现有巨型庆祝协调：继续消费自然完成事件，不关心专注和休息的视觉差异。

## 测试决策

- 测试应验证外部行为和模块契约，不测试私有实现细节。
- 计时偏好测试应覆盖默认值：专注 5 / 25 / 45、休息 5、两条默认文案、休息声音开启。
- 计时偏好测试应覆盖旧 settings 数据缺少专注计时配置时能自动使用默认值。
- 计时偏好测试应覆盖时长范围必须为 1 到 180 分钟。
- 计时偏好测试应覆盖专注时长不允许重复。
- 计时偏好测试应覆盖专注时长保存后升序。
- 计时偏好测试应覆盖空文案恢复默认。
- 计时偏好测试应覆盖文案去除首尾空白、压平换行和长度限制。
- 计时启动测试应覆盖专注计时和休息计时都能启动。
- 计时启动测试应覆盖同一时间只允许一个计时。
- 计时启动测试应覆盖计时启动时捕获完成文案和声音策略。
- 计时启动测试应覆盖设置变化不影响当前计时完成行为。
- 计时完成测试应覆盖专注结束使用专注文案且不主动播放声音。
- 计时完成测试应覆盖休息结束使用休息文案且在开关开启时播放系统声音。
- 计时完成测试应覆盖休息结束声音开关关闭时不播放声音。
- 计时完成测试应覆盖取消计时不会发送通知、声音或完成事件。
- 菜单测试应覆盖未计时时展示 3 个专注入口和 1 个休息入口。
- 菜单测试应覆盖专注中只展示“专注中：还剩 MM:SS”和取消入口。
- 菜单测试应覆盖休息中只展示“休息中：还剩 MM:SS”和取消入口。
- 菜单测试应继续覆盖剩余时间通过更新已有菜单项文本刷新，而不是每秒重建整棵菜单。
- 宠物事件测试应覆盖专注结束和休息结束都能进入现有巨型庆祝流程。
- 宠物事件测试应覆盖隐藏和拖拽中的完成规则保持不变。
- 设置页 smoke test 应覆盖修改 3 个专注时长、休息时长、两条文案和声音开关后重新打开仍然保留。
- 设置页 smoke test 应覆盖默认窗口打开时能访问“桌宠行为”“专注计时”和“活动区域”，右侧内容可以正常滚动。
- 设置页 smoke test 应覆盖分钟控件连续点击上下箭头不会选中文本，也不会导致按钮失效。
- 设置页 smoke test 应覆盖快速修改多个设置项时，后保存的字段不会覆盖先保存的字段。
- 桌面 smoke test 应覆盖专注完成、休息完成、休息完成声音、取消计时、隐藏完成和拖拽完成。
- 如果实现触及 Rust 侧设置、菜单或计时状态，应运行 Rust 测试。
- 如果实现触及前端设置页，应运行前端构建。
- 打包前的手工验收应确认菜单栏和宠物右键菜单行为一致。

## 进一步说明

- 这次允许“休息结束声音”是对旧边界的窄化调整：它属于休息结束提醒，不是宠物动作音效，也不是完整声音系统。
- 休息结束声音的最终落地方式是单独播放 macOS 内置 `Glass` 系统声音，原因是通知插件的声音参数在当前环境下不够可靠。
- 设置页交互属于本 PRD 的必要验收面：默认打开不能挤没专注计时，分钟控件不能因为连点而失效。
- 专注结束和休息结束都复用现有巨型庆祝，是为了避免把本轮扩成视觉素材或动画状态工作。
- 这轮最重要的产品判断是：Deskmon 可以帮助用户形成“专注一下、休息一下、回来继续”的轻节奏，但不替用户管理任务，也不替用户推进循环。
- 后续如果用户明确需要完整番茄能力，应作为新的 PRD 处理，不在本 PRD 内顺手扩展。
