## 问题陈述

Deskmon 已经能通过宠物动画、中央放大展示和 macOS 系统通知反馈应用内部事件，但外部本地工具没有一个稳定、轻量的入口来调用宠物完成提醒。Codex、构建脚本或其他自动化任务执行结束后，只能继续使用系统通知或弹窗；这类反馈和 Deskmon 的桌宠体验割裂，也无法让宠物成为桌面上的统一提示角色。

用户希望 Deskmon 提供一个本机命令行接口。外部工具传入一段短文字后，同一只宠物快速来到用户当前关注的屏幕中央，播放醒目的庆祝舞蹈，并用游戏式对话框传达消息。提醒应足够明显，但不能抢走键盘焦点、阻塞当前应用、播放声音或要求用户确认。

这个能力不是远程通知服务，也不是完整通知中心。V1 只解决“已运行且可见的 Deskmon 接收本机短消息，并让宠物进行一次临时提示”这一条主路径。Deskmon 未运行或宠物已隐藏时，调用应静默忽略；提醒不保存历史，不进入“记忆力”，也不改变现有专注、截图和系统通知规则。

## 解决方案

### 本地调用入口

Deskmon 提供 macOS 本地命令 `deskmon notify`。正文通过 `--text` 传入且为必填参数，标题通过 `--title` 传入且为可选参数。典型调用是 `deskmon notify --title "Codex" --text "任务执行完成"`。

公开契约只包含 CLI 命令、参数和退出行为。CLI 与正在运行的 Deskmon 通过当前 macOS 用户专属的本地进程间通信通道交换消息；底层协议不对外承诺，也不监听 TCP 端口。

调用时 Deskmon 未运行，CLI 不启动应用、不输出错误并返回成功。Deskmon 正在运行但宠物已隐藏时，应用接收后静默忽略，CLI 同样返回成功。提醒属于尽力而为的附加反馈，不能让 Codex、构建任务或自动化流程因为 Deskmon 状态而失败。

Deskmon 接收有效消息并纳入当前提醒状态后，CLI 立即退出，不等待动画播放完成。成功时默认不输出任何内容；只有命令参数格式无效，或已运行的 Deskmon 出现明确的内部接收错误时，才向标准错误输出简短原因并返回非零退出码。

### 文本契约

- 标题可选，最多显示 10 个完整 Unicode 字符。
- 正文最多显示 50 个完整 Unicode 字符。
- 超长内容自动截断，不报错；截断后的最后一个字符使用 `…`，省略号计入长度上限。
- 截断必须按用户感知的完整字符处理，不能切坏 emoji、组合字符或其他 Unicode 字素。
- 标题规范化为单行，连续空白压缩为一个空格。
- 正文允许最多 2 个换行，也就是最多 3 个自然段；连续空行压缩为一个换行。
- 正文规范化后为空时，应用直接忽略。即使存在标题，也不展示提醒；CLI 静默返回成功。
- 标题和正文只按纯文本渲染，不解析 Markdown、HTML、链接或其他富文本标记。
- 规范化与截断后的最终显示内容同时作为去重依据。

这个接口只用于普通状态提示。命令行参数可能短暂出现在进程列表或 shell 历史中，提醒文字也会直接显示在屏幕上，因此密码、令牌和其他敏感信息不应通过该接口传递。

### 中央宠物提醒

收到第一条有效提醒时，Deskmon 使用当前这只宠物进入临时外部提醒展示，不创建宠物分身，也不创建第二个独立宠物窗口。宠物的后台业务状态继续运行，但视觉上暂时切换到提醒表演。

普通状态下，提醒目标屏幕优先取鼠标指针所在屏幕；无法取得鼠标位置时回退到宠物当前所在屏幕，再无法匹配时回退到主显示器或第一个可用显示器。宠物本身尽量位于目标屏幕可用工作区的视觉中心，对话框根据剩余空间自适应布局。

进入提醒时，宠物约用 0.6 秒按中心点移动并放大到目标位置；结束后约用 0.5 秒恢复。进入和退出时长是首轮调校基线，后续可根据真实体验微调，不作为不可变常量。动效不能瞬移、弹跳、使用弧线路径或物理模拟。

提醒期间复用现有 `celebrate` 动画。宠物目标边长约为目标屏幕可用工作区短边的 35%，意图范围约 280px 到 460px，最终尺寸和位置必须限制在可用工作区内。尺寸同样属于首轮视觉基线，可在不改变产品语义的前提下调校。

提醒期间宠物临时置顶并跨 Space 显示，即使用户日常关闭“总在最前”，也要出现在普通窗口和全屏应用上方。提醒不得主动聚焦窗口、抢走当前应用的键盘输入或拦截鼠标。外部对话框和宠物提醒区域点击穿透；如果当前中央展示中包含原有专注操作按钮，只有这些原有按钮继续接收点击。

### RPG 对话框

外部提醒文字使用靠近宠物的 RPG 游戏对话框，不使用 macOS 通知中心卡片，也不把文字覆盖到宠物本体上。

对话框使用固定的深色半透明面板、浅色文字、像素化边框和轻微阴影，以便覆盖任意桌面背景时仍保持稳定对比度。文字使用系统中文字体，游戏感由面板、标题名牌和逐字动画表达，不额外打包像素字体。

存在标题时，标题显示为最多 10 字的小名牌；未传标题时不显示空名牌。正文显示在面板主体中，并以每字约 25–30ms 的较快速度逐字出现。标题立即显示。每条新增提醒只播放一次逐字效果，已经显示的提醒不会因为其他消息到达而重新逐字播放。

普通提醒优先把对话框纵向排列在宠物右侧；右侧空间不足时改到左侧，再不足时放到下方。宠物本身保持视觉居中，布局算法负责把最多 3 个对话框限制在目标屏幕的可用工作区内。

如果宠物已经处于专注完成、休息中或休息完成的中央交接状态，不重新移动或缩放宠物，也不覆盖已有专注对话框。外部提醒放到原对话框未占用的一侧；宠物临时切换到 `celebrate`，外部提醒结束后恢复与当前专注阶段一致的视觉状态。专注倒计时、状态机和操作按钮全程继续工作。

### 并发、去重与停留时间

外部提醒始终维护一个最多 3 条的实时对话列表，不使用多组轮播，也不让宠物在多组之间反复回位。

- 消息按到达顺序从上到下排列。
- 第 4 条不同消息到达时，移除当前最旧的一条，把新消息追加到列表底部。
- 10 秒内规范化与截断后的标题和正文完全相同，视为重复提醒。
- 重复提醒不增加新对话框，而是在现有条目上显示 `×2`、`×3` 等次数。
- 重复次数最多显示为 `×99+`，避免计数标签持续变宽。
- 重复提醒同样视为新事件，并重置整场表演的结束时间。

外部提醒固定停留 3 秒。第一条有效消息开始表演；之后每条新消息或重复计数更新都会把结束时间重置为“本次到达时刻 + 3 秒”。不设置额外总时长硬上限，因此只要持续收到消息，宠物就会持续在中央跳舞；最后一次新增或重复更新后的 3 秒结束后才恢复。

### 与现有交互并行

外部提醒不能改变专注计时、休息计时、自主移动暂停等后台状态。宠物处于普通状态时，视觉暂时切换为提醒；结束后根据当时最新业务状态恢复，而不是盲目恢复开始提醒时的旧 mood。

用户正在拖拽宠物时，外部提醒不强行夺走窗口。提醒暂存在最多 3 条的实时列表中，松手后立即展示。Deskmon 正在进行区域截图或原地标注时也延迟展示，避免宠物突然移动、对话框遮挡选区或进入最终截图。延迟期间仍执行最多 3 条、第四条替换最旧和重复合并规则。

因拖拽或截图延迟的提醒，其停留时间从真正显示时开始计算，而不是从消息到达时开始，避免解除保护状态后提醒刚出现就消失。

用户只是暂停宠物自主移动时，外部提醒仍然正常展示；结束后恢复暂停状态。用户在提醒期间从菜单栏隐藏宠物时，直接操作优先：立即中止提醒、清空尚存消息、恢复普通尺寸后隐藏宠物。

提醒开始时记录恢复所需的位置语义，但临时尺寸、位置和置顶状态不能写入用户设置。提醒期间如果用户修改宠物尺寸或置顶偏好，结束时使用最新偏好；位置回到原宠物中心附近，并根据当前可见工作区重新限制，保证宠物不会因显示器变化而落到屏幕外。

### CLI 安装与卸载

安装 `.app` 本身不会静默修改终端环境。Deskmon 设置页提供独立的“命令行工具”面板，由用户主动安装、更新或卸载。安装过程通过一次明确的 macOS 管理员授权，在 `/usr/local/bin/deskmon` 建立指向应用包内 CLI 的符号链接。

使用符号链接可以让 Deskmon 覆盖升级后自动使用应用包内的新 CLI，不要求用户每次更新后重新安装。应用被移动到其他位置后，用户可以从设置页更新或重新安装以修复链接。

如果目标命令已经指向当前 Deskmon 或旧版 Deskmon，可以安全更新；如果同名路径属于其他文件或程序，安装必须中止并明确提示冲突，不得直接覆盖。

安装成功后，Deskmon 自动播放一次本功能的测试提醒：标题为“Deskmon”，正文为“命令行工具已安装”。不额外提供长期存在的测试菜单项。

设置面板持续展示未安装、已安装、需要更新或路径冲突四种状态，并显示固定安装路径。安装完成后操作切换为“卸载…”。卸载同样经过管理员授权，只移除属于当前 Deskmon 的命令链接，不卸载应用、不删除设置，也不能删除同名的其他程序。等待授权、成功与失败反馈都显示在设置面板内；路径冲突时不得提供覆盖动作，只允许用户处理冲突后重新检查。

Deskmon 不自动修改 Codex、shell、项目或自动化配置。产品只提供 CLI 契约和集成示例；用户可以在具体项目规则、技能或自动化流程中决定何时调用。

73. 作为用户，我不需要成功、警告和错误等级，以便调用契约保持简单。
74. 作为用户，我不想外部提醒保存历史，以便短暂提示不会变成新的管理负担。
75. 作为用户，我不想提醒进入“记忆力”，以便通知文字不会扩散到剪贴板数据域。
76. 作为用户，我不想应用重启后恢复旧提醒，以便过期消息不会再次出现。
77. 作为用户，我不需要额外的外部提醒设置开关，以便设置页保持精简。
78. 作为用户，我想通过主动安装 CLI 启用集成，以便 Deskmon 不擅自修改终端环境。
79. 作为用户，我接受安装 CLI 时的一次管理员授权，以便命令能稳定进入 PATH。
80. 作为用户，我想 Deskmon 更新后 CLI 自动使用新版本，以便不用反复安装。
81. 作为用户，我想安装程序保护已有同名命令，以便不会覆盖其他工具。
82. 作为用户，我想安装成功后看到测试提醒，以便立即确认整条链路可用。
83. 作为用户，我想从设置页卸载 CLI，以便集成是可逆的。
84. 作为用户，我想卸载 CLI 不影响应用和设置，以便只移除终端入口。
85. 作为 Codex 用户，我想获得一段调用示例，以便能在项目规则中自行接入。
86. 作为用户，我不想 Deskmon 自动修改 Codex 配置，以便每个项目仍能决定何时提醒。
87. 作为开发者，我想本地通信只允许当前用户访问，以便不暴露网络服务。
88. 作为开发者，我想底层通信协议保持私有，以便未来可以替换实现而不破坏调用方。
89. 作为开发者，我想提醒规则集中在可测试状态模块中，以便并发、去重和计时不依赖真实窗口。
90. 作为开发者，我想展示状态与普通宠物设置分离，以便临时位置和尺寸不会持久化。
91. 作为开发者，我想布局算法可以独立测试，以便多显示器、交接框和 3 条消息的位置稳定。
92. 作为开发者，我想 CLI 安装逻辑验证链接归属，以便卸载和更新不会误删其他文件。
93. 作为开发者，我想提醒与截图任务协调，以便截图工具和最终 PNG 都不包含外部提示。
94. 作为开发者，我想提醒与专注状态机解耦，以便新增外部入口不改变现有计时语义。

## 实现决策

- V1 仅支持 macOS 本机调用，不实现 Windows、Linux、局域网或互联网入口。
- 公开接口固定为 `deskmon notify`，使用可选 `--title` 和必填 `--text`；不支持位置参数、stdin、JSON 或批量输入。
- 外部调用只依赖 CLI；当前用户专属的本地 IPC 是内部实现，不作为兼容性承诺。
- 本地 IPC 必须使用用户隔离的端点和严格权限，不能监听 TCP 端口，也不需要额外令牌。
- CLI 不能启动未运行的 Deskmon；无法连接到未运行实例时静默返回成功。
- 应用已运行但宠物隐藏时，消息静默忽略并返回成功。
- 应用接收有效消息后立即确认，CLI 不等待展示完成。
- 成功、未运行、隐藏和空正文均无标准输出并返回成功；真正的参数格式或内部接收错误才输出到标准错误并返回非零。
- 标题上限 10 个字素，正文上限 50 个字素；超长安全截断并在限制内追加省略号。
- 标题压成单行；正文最多保留 2 个换行；正文为空时整条忽略。
- 标题与正文必须作为纯文本处理，不能进入 HTML、Markdown 或链接解析路径。
- 去重基于规范化和截断后的最终标题与正文，时间窗口固定为 10 秒。
- 重复次数随条目展示，视觉上限为 `×99+`。
- 提醒状态最多保存 3 条；第 4 条不同消息替换最旧条目，不建立额外轮播队列。
- 每次新增或重复更新都重置结束时间；结束时间固定为本次到达时刻加 3 秒。
- 持续消息允许让提醒持续存在，不增加总时长硬上限。
- 外部提醒必须使用当前宠物，不创建宠物分身或第二套角色状态。
- 普通状态下目标屏幕优先使用鼠标所在屏幕，宠物自身保持视觉居中，对话框自适应换边。
- 进入约 0.6 秒、退出约 0.5 秒、目标短边约 35%、目标尺寸约 280–460px，均为可调视觉参数，不是持久设置。
- 提醒复用现有 `celebrate` 素材，不为 V1 新增通知专用 sprite 状态。
- RPG 对话框使用固定深色高对比面板、像素边框、系统字体和快速逐字动画。
- 外部提醒临时置顶、跨 Space、允许覆盖全屏应用，但不能主动聚焦或拦截指针。
- 外部对话框不提供按钮、链接、关闭动作或确认流程；到时自动消失。
- 专注中央交接存在时保留原布局和操作，把外部提醒放到空闲一侧，不改变专注状态机。
- 拖拽和截图任务是两个硬保护状态；消息延迟到状态结束后展示，显示计时也从那时开始。
- 自主移动暂停不阻止提醒；提醒结束后继续保持暂停。
- 用户隐藏宠物会立即中止提醒、清空消息并恢复普通展示后隐藏。
- 临时位置、尺寸和置顶状态不能写入设置；恢复时使用最新尺寸与置顶偏好，并把原位置限制到当前可见工作区。
- 外部提醒不播放声音、不发送 macOS 系统通知、不持久化、不进入“记忆力”或应用日志。
- V1 不提供提醒等级、操作按钮、历史窗口、独立设置开关、减少动态效果适配或 macOS 专注模式感知。
- Deskmon 现有专注完成、截图保存和“记忆力”反馈不迁移到外部提醒系统。
- CLI 通过设置页的“命令行工具”面板显式安装、更新和卸载，安装目标为 `/usr/local/bin/deskmon`，使用管理员授权创建或移除属于 Deskmon 的符号链接。
- 设置页展示未安装、已安装、需要更新和路径冲突四种状态；操作期间禁用重复提交，并把等待授权、成功和失败反馈留在面板内。
- 安装前必须验证现有目标归属；只能更新 Deskmon 自己的链接，不能覆盖或删除其他同名命令。
- 安装成功后通过内部同一入口展示一次固定测试提醒，不新增测试菜单项。
- Deskmon 只提供 Codex 集成示例，不自动修改 Codex、项目规则、技能或 shell 配置。

建议模块边界：

- CLI 客户端与本地 IPC 适配器：解析公开参数、连接当前用户的 Deskmon 实例、映射退出码，不感知宠物展示细节。
- 外部提醒状态模块：负责文本规范化、Unicode 截断、去重计数、最多 3 条替换、停留时间和硬保护期间的延迟状态。
- 宠物提醒展示协调器：协调普通宠物、专注中央交接、隐藏、暂停、拖拽、截图、显示器变化和最终恢复。
- RPG 对话框与布局模块：负责最多 3 个对话框、名牌、逐字状态、屏幕边界和与专注对话框的对侧布局。
- CLI 安装管理器：负责管理员授权、符号链接归属检查、安装、更新、卸载和安装后的测试提醒。

## 测试决策

- 测试只验证用户可见行为和深模块契约，不依赖具体 DOM 层级、socket 文件名或私有函数结构。
- 文本规范化测试覆盖标题单行、正文最多 2 个换行、连续空白、空正文和标题存在但正文为空。
- Unicode 截断测试覆盖中文、emoji、组合字符、标题 10 字边界、正文 50 字边界和省略号计入上限。
- 纯文本测试验证 Markdown、HTML 和 URL 只显示原始字符，不生成富文本或可点击内容。
- 去重测试覆盖 10 秒内相同内容合并、窗口外不合并、规范化后相同、截断后相同和不同标题不合并。
- 重复计数测试覆盖 `×2`、持续递增、`×99+` 上限和重复事件重置结束时间。
- 容量测试覆盖 1–3 条到达顺序、第 4 条替换最旧、替换后新消息位于底部和无额外轮播队列。
- 停留时间测试覆盖短消息和长消息均固定 3 秒、新消息重置、最后一条决定结束和持续消息无限续期。
- 硬保护测试覆盖拖拽期间延迟、松手后展示、截图期间延迟、截图结束后展示和延迟期间仍遵守容量与去重规则。
- 延迟计时测试验证停留时间从真正展示开始，而不是从最初到达开始。
- 展示协调测试覆盖普通状态进入、`celebrate` 切换、结束后按最新业务状态恢复和自主移动暂停保持。
- 专注并行测试覆盖专注完成、休息中、休息完成三个中央阶段，确保外部对话位于另一侧、按钮可用且状态机不变。
- 隐藏测试覆盖提醒前隐藏静默忽略、提醒期间隐藏立即中止、消息清空、普通尺寸恢复和重新显示后不重播。
- 恢复测试覆盖提醒期间修改尺寸、修改置顶、显示器移除、工作区变化和最终位置 clamp。
- 布局测试覆盖鼠标屏幕优先、宠物屏幕回退、主屏回退、负坐标显示器、不同缩放比例和可用工作区边缘。
- 对话框布局测试覆盖右侧、左侧、下方、1–3 条纵向排列、专注框占右侧或左侧以及宠物保持视觉居中。
- 逐字状态测试覆盖标题立即显示、正文按字出现、新条目单独播放和旧条目不重播。
- 指针与焦点 smoke test 验证普通应用、文本输入和全屏应用中提醒不抢焦点且点击穿透。
- Space smoke test 验证提醒出现在鼠标所在显示器与当前全屏 Space，结束后宠物回到原位置附近。
- CLI 单元测试覆盖帮助、必填参数、可选标题、未知参数、空正文成功和超长内容成功截断。
- CLI 集成测试使用临时本地端点覆盖应用未运行成功、接收成功立即退出、明确内部错误返回非零和默认无输出。
- IPC 权限测试验证其他 macOS 用户不能连接，不创建网络监听，也不把底层协议暴露为公共兼容契约。
- 安装管理测试覆盖目标不存在、当前链接、旧版 Deskmon 链接、其他同名文件冲突和只删除自身链接。
- 设置页测试覆盖四种安装状态、安装/更新/卸载按钮、授权等待状态、路径冲突重新检查和内联错误反馈。
- 手工安装测试验证管理员授权、`deskmon` 在新终端中可用、应用升级后仍可用、应用移动后可重新安装修复。
- 手工卸载测试验证只移除 CLI，不影响应用、设置或其他同名程序。
- 安装反馈测试验证成功后显示“Deskmon / 命令行工具已安装”，且不会同时发送系统通知或声音。
- 回归测试验证 Deskmon 原有专注系统通知、截图保存通知、“记忆力”反馈、宠物拖拽和自主移动行为保持不变。
- 手工长时间测试验证频繁消息、持续重复、不断替换最旧条目和最终停止发送后都能稳定恢复。
- 测试结构沿用现有项目先例：纯状态与几何算法使用确定性单元测试，Tauri 编排使用 Rust 行为测试，跨 Space、焦点、全屏和管理员授权使用 macOS 手工 smoke test。

## 不在范围内

- Windows、Linux、iOS、Android 或 Web 实现。
- 局域网、互联网、HTTP、WebSocket、远程推送或云端消息中继。
- 对外公开底层 Unix Socket 路径、消息协议或 SDK。
- 自动启动、登录项启动或 CLI 调用时拉起 Deskmon。
- 自动安装 CLI、首次启动修改 PATH、修改 `.zshrc` 或其他 shell 配置。
- 自动修改 Codex 全局配置、项目规则、技能或任务结束行为。
- stdin、JSON、批量文件、管道流式输入或长连接客户端。
- Markdown、HTML、富文本、链接、图片、图标、附件或自定义字体。
- 密码、令牌、私密变量或其他敏感信息的安全传输与遮罩展示。
- 成功、警告、错误、进度等提醒等级和对应颜色。
- 操作按钮、链接打开、确认、取消、延后提醒或用户输入。
- 点击关闭、键盘关闭、悬停暂停或手动延长。
- 声音、铃声选择、系统提示音或自定义音频。
- 同时发送 macOS 系统通知。
- 提醒历史、通知中心、搜索、归档、已读状态或跨重启恢复。
- 写入系统剪贴板、“记忆力”、日志、数据库或用户设置。
- 独立外部提醒设置开关、来源授权名单或调用令牌。
- macOS 专注模式、勿扰模式或屏幕共享状态感知。
- `prefers-reduced-motion` 或其他减少动态效果适配。
- 宠物分身、多个宠物窗口或每条消息一只宠物。
- 通知专用 sprite、皮肤、动作选择或外部自定义动画。
- 把 Deskmon 内部专注、截图和“记忆力”反馈迁移到新提醒系统。
- App Store 沙箱发行下 CLI 安装和本地 IPC 的适配。

## 进一步说明

- V1 的核心价值是“让同一只宠物成为本地自动化的短消息传达者”，不是构建通用通知平台。
- CLI 是唯一公开兼容层；内部通信、窗口拆分和状态实现可以在保持外部行为不变的前提下调整。
- “并行”指最多 3 条消息在同一场宠物表演中共同展示，不是创建宠物分身，也不是让单个宠物同时处于两个视觉位置。
- 0.6 秒进入、0.5 秒退出、35% 目标比例、280–460px 尺寸和 25–30ms 逐字速度都是首轮实现基线，应通过真实桌面 smoke test 调校。
- 宠物隐藏是明确的安静状态；自主移动暂停不是静音状态；拖拽和截图只是短暂硬保护状态。
- 如果未来要让 Deskmon 内部事件也使用游戏对话提醒，应另开需求统一通知策略，不在本 PRD 中顺带迁移。
