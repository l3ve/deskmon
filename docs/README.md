# Deskmon 文档导航

这里保存产品需求、设计资料、验收清单和尚未立项的想法。当前产品能力以仓库根目录的 [README](../README.md) 和代码为准。

## 需求文档

[PRD 索引](prds/README.md) 记录 001–011 的实现状态与覆盖关系。

PRD 是需求形成时的决策记录。后续 PRD 可以覆盖旧方案，因此不能只读某一份历史 PRD 来判断当前行为。

## 设计资料

`designs/` 保存“记忆力”窗口和变量库的设计稿、流程图及说明。

设计稿用于解释信息架构和视觉方向。若设计稿与当前代码不同，以已实现代码、最新 PRD 和 README 为准。

## 验收清单

[V1 Smoke Checklist](qa/v1-smoke-checklist.md) 用于打包安装后的 macOS 手工回归。

自动化测试负责状态和边界，桌面窗口、原生菜单、权限、拖拽与视觉表现仍需要在真实 macOS 环境中验证。

## 想法收件箱

[想法收件箱](ideas.md) 只保存尚未成熟的方向，不代表承诺或排期。

想法经确认后再建立独立 PRD；实现细节和长期工程决策不写入想法列表。

## 工程记录

- [工程日志索引](../.ai/logs/README.md)：按日期保存每轮目标、完成情况和修改文件。
- [长期关键决策](../.ai/keypoint.md)：记录关键文件、稳定决策、已知问题和避坑规则。

工程日志是历史事实，不回写成当前产品说明。需要理解现状时，先看 README 和 PRD 索引，再按需追溯具体日期日志。
